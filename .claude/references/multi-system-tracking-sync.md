# 多系統追蹤同步原則

多步驟流程的進度必須在多個追蹤系統中同步記錄，確保從任何入口都能了解全貌。

---

## 核心原則

任何跨多步驟的流程（Legacy 接手、版本發布、大型重構等），進度必須同時存在於：

| 系統 | 角色 | 記錄什麼 |
|------|------|---------|
| **核心文件** | 流程詳細記錄 | 每步驟的狀態、產出物、commit hash |
| **Worklog** | 版本級進度追蹤 | 流程背景（原因、路徑）+ 進度事件記錄 |
| **Ticket** | 任務級執行追蹤 | 每步驟（或合併步驟）的執行歷程 |

## 同步規則

### Worklog 記錄要求

在版本 worklog 中記錄流程背景：
- 啟動原因和日期
- 核心文件路徑（指向詳細記錄）
- 當前進度摘要

步驟完成事件記錄在「進度追蹤」區段：
```
- YYYY-MM-DD: {流程名稱} 步驟 N 完成 -- [摘要]
```

### Ticket 記錄要求

| 策略 | 適用場景 | 說明 |
|------|---------|------|
| 每步驟一個 Ticket | 步驟工作量大 | 標題包含流程名稱和步驟編號 |
| 合併小步驟 | 步驟簡單可合併 | 標題明確包含所有步驟編號 |
| 子 Ticket | 步驟內需拆分 | 父 Ticket 為步驟，子 Ticket 為子任務 |

**Ticket 建立時機**：步驟**開始前**建立，不是完成後補建。

### 禁止行為

- 只在核心文件記錄，不建立 Ticket
- Ticket 標題不包含流程/步驟標識
- 步驟完成但未在 worklog 進度追蹤中記錄

## 適用場景

- Legacy Code 接手流程（詳見 `.claude/skills/doc/references/legacy-code-workflow.md`）
- 版本發布流程（詳見 `.claude/skills/version-release/SKILL.md`）
- 跨版本大型重構

### 核心發現

**ARCH-015 實際分界線不是「豁免清單 vs 非豁免」，而是「是否為主 repo 內」**：
- 主 repo 內 .claude/（包含 agents/rules 所有子目錄）→ subagent 全通行（Write + Edit）
- 外部 worktree 內 .claude/ → subagent 全阻擋

### 對 ARCH-015 原矩陣的更新建議

原矩陣 row 「主 repo cwd → worktree 內 .claude/ → 拒絕」 正確
但反向測試「worktree cwd → 主 repo 內 .claude/」：本實驗證實「成功」

真實規則推測：**target 路徑是否在主 session repo 樹內**
- 在主 repo 樹內 → 允許
- 不在主 repo 樹內 → 拒絕

### 對 PC-092 落地的影響

重大影響！W5-047 原設計「.claude/ 修改必須在主 repo cwd 派發」可能過度限制：
- 實際上從 worktree cwd 也能修改主 repo /.claude/
- W5-049 的 src/ worktree-first 規則仍正確
- agent-dispatch-validation-hook 可能 over-restrictive，需重新設計

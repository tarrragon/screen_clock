# 決策者快速參考清單（PM Checklist）

WRAP 決策框架的每階段自問清單。在決策時快速掃描使用。

---

## 快速模式（5 分鐘）

適用：連續失敗、被困住、宣稱「做不到」

```
[錨點] 誰是客戶？當前版本核心目標？
   → 這個問題影響核心目標嗎？否 → 建任務延後

[W] 我有幾個選項？
   → 只有 1 個？→ 強制再找 2 個
   → 搜尋社群了嗎？→ 沒有 → 先搜尋再決定
   → 選項有實質差異嗎？→ 沒有 → 假選項，重新擴增

[R 核心] 這類問題的常見解法和成功率？（基本率 > 預測）
   → 問過去，不問未來

[A 前置] 框架概括測試：能否用一句話概括所有方案？
   → 能 → 回 W 擴增多元選項
   → 否 → 進 A 核心

[A 核心] 投入這個問題的時間，會擠壓哪個更重要的目標？（機會成本）
   → 列出放棄什麼

[決定] 選項消失測試：以上選項全部被移除，還能怎麼做？
```

---

## 完整模式（15-30 分鐘）

適用：偏離核心、重大決策

### 0. 錨點確認

- [ ] 誰是我們的客戶？
- [ ] 當前版本的核心目標是什麼？
- [ ] 這個問題影響核心客戶/核心目標嗎？
- [ ] 如果不影響：建提案/任務延後，回到核心任務

### W — 擴增選項

- [ ] 我有幾個選項？（至少 3 個有實質差異的）
- [ ] 當前專案有沒有類似解法？（grep / LSP）
- [ ] 社群有沒有人解決過？（GitHub Issues / 文件）
- [ ] 其他領域有沒有類比方案？（AI 知識 / 網路搜尋（WebSearch））
- [ ] 選項消失測試：以上選項全部被移除，還能怎麼做？
- [ ] 假選項偵測：選項之間是不同途徑，還是同一途徑的參數微調？

### R — 現實檢驗

- [ ] 這類問題的常見解法有哪些？成功率？（基本率）
- [ ] 需要什麼事證才能證明這個方法可行？（事證問題）
- [ ] 有沒有辦法用最小成本驗證假設？5 分鐘內能得到答案嗎？（試水溫）
- [ ] 搜尋了具體案例嗎？讀了內文不只標題？（近距離檢視（Zoom In））
- [ ] 最強版本論證（Steelman）：能闡明被放棄選項的優點嗎？能列出選定方案的 3 個缺點嗎？

### A — 拉開距離

**前置強制檢查**：

- [ ] 框架概括測試：能否用一句話概括所有候選方案？「能」→ 選項多元性不足，回 W
- [ ] 反面框架識別：問題的反面框架是什麼？（未列出即強制停下列出）
- [ ] 反向思考（Consider the Opposite）：若我相信的正好相反會怎樣？能提出有力相反論述 → 重新定義問題後從 W 重跑

**正式 A 階段**：

- [ ] 每個選項旁列出機會成本了嗎？
- [ ] 這個決策服務於哪個優先事項？
- [ ] 投入這個問題的時間，會擠壓哪個更重要的目標？
- [ ] 不值得投入的項目，是否已記錄到提案系統或任務延後？

### P — 準備好犯錯

- [ ] 行前預想：假設失敗了，列出 3 個最可能原因（任一 > 50% → 重新評估）
- [ ] 未來區間：最好結果？最壞結果？兩端都能接受？
- [ ] 安全係數：預估時間 x 1.3-1.5
- [ ] 回退計畫：失敗怎麼回退？回退成本是多少？

### 5. 絆腳索設定

- [ ] 設定期限：非核心問題最多花多少分鐘？
- [ ] 設定失敗門檻：幾次失敗後切換方向？
- [ ] 設定偏離警報：什麼信號代表已偏離核心目標？

## 決策品質自測

完成 WRAP 後，用以下問題自測：

| 問題                           | 通過標準 |
| ------------------------------ | -------- |
| 我考慮了幾個有實質差異的選項？ | >= 3 個  |
| 我搜尋了社群/文件嗎？          | 是       |
| 我能闡明反對意見嗎？           | 是       |
| 我列出了機會成本嗎？           | 是       |
| 我做了行前預想嗎？             | 是       |
| 我有回退計畫嗎？               | 是       |

4/6 以上通過 → 決策品質足夠。
3/6 以下 → 回到對應階段補充。

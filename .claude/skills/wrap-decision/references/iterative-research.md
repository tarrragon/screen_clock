# 多輪迭代查詢方法論

> **本檔位置**：W 階段 + R 階段擴充。WRAP 既有的「擴增選項」與「現實檢驗」原則本身不規範**搜尋的迭代結構**，本檔補充。
>
> **來源實證**：規則設計過程中，4 輪查詢累積 27 個洞察，第 4 輪反向驗證揭露核心悖論。

---

## 為什麼一次性搜尋不夠

| 失敗模式                         | 後果                             |
| -------------------------------- | -------------------------------- |
| 第一輪關鍵字基於「目前理解」設定 | 找到的多是「已知方向的支援證據」 |
| 不做反向搜尋                     | 落入確認偏誤                     |
| 不做精準化深挖                   | 停留在表面案例，缺學術深度       |
| 不做用戶介入點記錄               | 用戶補充的盲點訊息流失           |

---

## 四輪結構

| 輪次 | 名稱               | 關鍵字策略                     | 目的                               | 對應 WRAP               |
| ---- | ------------------ | ------------------------------ | ---------------------------------- | ----------------------- |
| 1    | 發散型 (Divergent) | 廣泛關鍵字鋪面，5-8 個不同領域 | 建立問題地圖，找出所有可能相關方向 | W 擴增初步              |
| 2    | 具體化 (Concrete)  | 基於第一輪發現深入特定主題     | 將抽象主題轉為具體案例             | W 擴增深化              |
| 3    | 精準化 (Precise)   | 從前兩輪提煉精準關鍵字深挖     | 找到最高質量的學術來源與案例細節   | R 現實檢驗              |
| 4    | 反向驗證 (Inverse) | 每個結論找反例 / 批評 / 反駁   | 防止確認偏誤，揭露悖論             | R 現實檢驗 + A 拉開距離 |

---

## 進入下一輪訊號

至少 5 條判定訊號，任一觸發即可進入下一輪：

1. 該輪新洞察 < 上一輪 50% → 邊際效益遞減（自然進入下輪或收斂）
2. 已涵蓋正反雙面 → 可進入收斂
3. 用戶補充重大反向觀察 → 立即進入反向驗證輪
4. 發現自我參照悖論 → 進入元層次反思輪
5. 出現結論互相衝突 → 進入優先級裁決輪

---

## 邊界條件

- **不一定要 4 輪**：簡單議題可能 2 輪夠
- **反向驗證輪幾乎不可省略**（確認偏誤防護）
- **每輪 8 個搜尋是 網路搜尋（WebSearch）平行查詢上限的經驗值**——可調整
- **每輪後必暴露偏好**（不偽裝中立），讓用戶可介入糾正方向

---

## 反向驗證實踐範本（第 4 輪標準格式）

### 標準表格格式

```
| 我們的結論 | 反向搜尋目標 |
|----------|-----------|
| [結論 1] | [批評 / 反例 / 反駁 / 限制] |
| [結論 2] | [批評 / 反例 / 反駁 / 限制] |
```

### 8 種反向方向類型

每輪反向搜尋必涵蓋至少 4 種，理想涵蓋全部 8 種：

| 反向類型                           | 識別問題                                         | 範例                                                              |
| ---------------------------------- | ------------------------------------------------ | ----------------------------------------------------------------- |
| 1. 方法論的學術批評                | 此方法論本身有什麼學術反駁？                     | MI 缺乏理論基礎、Voss 不適用長期關係                              |
| 2. 關鍵案例的反方論點              | 案例的另一方怎麼說？                             | 冤案的警方/檢察官立場                                             |
| 3. 結論的失效情境                  | 在什麼情境此結論失效？                           | MI 對強制 client 失效                                             |
| 4. 數字 / 統計的方法論限制         | 數字背後的方法論有什麼限制？                     | DarkBench 30-61% 的 sampling 限制                                 |
| 5. 普適性的文化 / 情境限制         | 此結論在不同文化 / 情境是否成立？                | Voss 不適用長期關係、自主性（autonomy）是西方偏見                 |
| 6. 隱性家長主義（paternalism）警示 | 此規則是否本身違反它要保護的價值？               | 保護用戶自主性（autonomy）的規則限制了決策者自主性（PM autonomy） |
| 7. 取捨揭露（trade-off）           | 此方案的隱藏代價是什麼？                         | 迎合（sycophancy）提升滿意度但傷害福祉                            |
| 8. 自我參照悖論識別                | 此規則的「正確示範」是否本身違反它要禁止的模式？ | 反窄框架（narrow framing）本身就是窄框架（narrow framing）        |

## 實證統計

| 輪次        | 搜尋數 | 新洞察數 | 累積洞察 | 關鍵突破                                           |
| ----------- | ------ | -------- | -------- | -------------------------------------------------- |
| 1（發散）   | 8      | 6 (A-F)  | 6        | Calibrated Questions 雙面性、Confirmshaming 對應   |
| 2（具體化） | 8      | 6 (G-L)  | 12       | Lifton 8 條件、MI OARS、Anthropic 自承權力不對等   |
| 3（精準化） | 8      | 7 (M-S)  | 19       | Anthropic 迎合（sycophancy）自證、DarkBench 30-61% |
| 4（反向）   | 8      | 8 (U-BB) | **27**   | **規則 5 是家長主義（paternalism）悖論揭露**       |

**關鍵觀察**：第 4 輪反向驗證的洞察數最高（8 個），且質量最尖銳（揭露核心悖論）。**反向輪是深度查詢的主軸**。

## 用戶介入點記錄

多輪查詢過程中，用戶補充常常打到決策者盲點。必記錄：

| 介入時機   | 介入內容                     | 引發效應                                         |
| ---------- | ---------------------------- | ------------------------------------------------ |
| 第 1 輪後  | 補充權力不對等 4 例          | 暴露決策者窄框架（narrow framing） → 第 2 輪轉向 |
| 規則設計中 | 指出「正確示範」也是強勢框架 | 機制設計元層次升級                               |
| 第 3 輪後  | 要求做反向搜尋               | 揭露核心悖論                                     |

**用戶補充是設計者盲點的最強發現器**。記錄這些介入點作為未來相似議題的提醒。

## 反模式

| 反模式       | 症狀                          | 修正                          |
| ------------ | ----------------------------- | ----------------------------- |
| 一輪定論     | 第一輪結果就下結論            | 至少做到第 2 輪               |
| 跳過反向驗證 | 直接從第 3 輪進入收斂         | 反向輪不可省略                |
| 確認式關鍵字 | 第 4 輪關鍵字仍偏向支持原結論 | 必須用「批評 / 反駁」明確反向 |
| 忽略用戶補充 | 用戶補充被當「離題」          | 用戶補充常常是核心訊號        |
| 偽裝中立     | 每輪後不暴露偏好              | 必暴露偏好讓用戶可介入        |

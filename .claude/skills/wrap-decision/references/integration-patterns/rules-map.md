# Rules Map Pattern

規則庫映射模式的責任是說明 WRAP 與專案規則庫如何分工。WRAP 提供決策品質框架；專案規則庫提供特定工作流的可執行細節。

---

## 分工

| 層級                | 責任                                             |
| ------------------- | ------------------------------------------------ |
| WRAP skill          | 定義何時要擴增選項、驗證假設、拉開距離、準備回退 |
| Integration pattern | 提供可攜模板，例如三問、觸發條件、來源核對       |
| Project rule store  | 定義專案內的實際指令、路徑、owner、狀態欄位      |
| Automation          | 依專案規則執行提醒、檢查或阻擋                   |

---

## 建議索引欄位

| 欄位            | 責任                                                                |
| --------------- | ------------------------------------------------------------------- |
| Rule name       | 規則名稱                                                            |
| WRAP stage      | 對應錨點確認（Anchor）/ Step 0 / W / R / A / P / 絆腳索（Tripwire） |
| Trigger         | 何時讀這條規則                                                      |
| Source of truth | 哪份文件或設定是權威來源                                            |
| Automation      | 是否有自動提醒或檢查                                                |
| Owner           | 誰負責更新                                                          |

---

## 維護原則

1. WRAP skill 不引用專案規則庫。
2. 專案規則庫可以引用 WRAP skill 與 integration patterns。
3. 同一條規則只保留一個權威來源。
4. 自動化工具讀權威來源，不複製清單。
5. 規則變更時，同步更新範例與反模式。

---

## 反模式

| 反模式                     | 修正                                        |
| -------------------------- | ------------------------------------------- |
| 核心 skill 寫入專案路徑    | 移到 project rule store                     |
| 多份文件重複觸發條件       | 建立單一真實來源（Source of Truth）與映射表 |
| 自動化行為與文字規則不同步 | 讓自動化讀設定來源                          |
| 規則只寫禁止，缺替代路徑   | 補下一步與升級條件                          |

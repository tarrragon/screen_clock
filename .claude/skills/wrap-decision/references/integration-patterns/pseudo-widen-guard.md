# 偽擴增選項防護（Pseudo Widen Guard）

偽擴增選項（pseudo-Widen）防護的責任是確認選項多元性發生在假設層級，而不只是實作手段層級。它特別適用於根因分析、架構決策與規則設計。

---

## 核心判準

真正的擴增選項（Widen）會提出不同根因假設或不同達成路徑。偽擴增選項（pseudo-Widen）只是在同一假設下列出多個變體，表面上有選項，實際上仍被單一框架鎖住。

---

## 四步驟

### 1. 寫出目前假設

```text
我目前假設問題是由 ______ 造成。
```

### 2. 列出相反假設

```text
如果問題由其他原因造成，最可能是什麼？
```

### 3. 檢查選項層級

| 檢查     | 通過標準                                 |
| -------- | ---------------------------------------- |
| 根因多元 | 選項來自不同根因假設                     |
| 路徑多元 | 選項包含新增、改造既有、零工具或流程調整 |
| 證據多元 | 每個選項需要不同證據驗證                 |
| 回退多元 | 每個選項的失敗與回退方式不同             |

### 4. 現實檢驗（Reality Test）

每個候選假設至少要有一個可驗證訊號。缺少可驗證訊號的選項保留為待驗假設，並在補齊證據後再進入方案比較。

---

## 警告信號

| 信號                           | 意義               |
| ------------------------------ | ------------------ |
| 所有選項能用一句話概括         | 選項仍在同一框架內 |
| 所有選項只差工具或參數         | 還沒碰到根因層     |
| 所有選項需要同一份證據         | 證據路徑缺乏多元性 |
| 被使用者重述問題後整份分析失效 | 原問題框架太窄     |

---

## 可複製提示

```text
先做偽擴增選項（pseudo-Widen）檢查：
1. 我現在假設的根因是什麼？
2. 相反假設是什麼？
3. 目前選項是否都接受同一根因？
4. 哪個證據能讓我放棄目前最相信的假設？
```

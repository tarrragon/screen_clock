# Simplified Three Questions

簡化三問的責任是在任務啟動前保留 WRAP 的最低品質門檻。它取 W/A/P 三個階段，讓小型任務能快速開始，也讓高風險任務及早升級完整 WRAP。

---

## 三問

| 階段 | 問題                                       | 目的                 |
| ---- | ------------------------------------------ | -------------------- |
| W    | 除了第一個方案，還有哪兩條可行路徑？       | 避免預設選項自動勝出 |
| A    | 這會擠壓哪個更重要的目標？                 | 顯性化機會成本       |
| P    | 如果完成後仍失敗，最可能的三個原因是什麼？ | 提前建立防護與回退   |

---

## 填答格式

```text
W：
- 首選路徑：
- 替代路徑 A：
- 替代路徑 B：
- 選擇理由：

A：
- 預估投入：
- 被擠壓目標：
- 現在做的理由：

P：
- 失敗原因 1 / 防護：
- 失敗原因 2 / 防護：
- 失敗原因 3 / 防護：
```

---

## 適用範圍

| 任務       | 適用方式                  |
| ---------- | ------------------------- |
| 小型實作   | 三問通過即可開始          |
| 文件更新   | 三問加既有內容搜尋        |
| 規則修改   | 三問加悖論檢查            |
| 分析與提案 | 三問只做啟動，接完整 WRAP |

---

## 升級條件

| 條件                             | 動作              |
| -------------------------------- | ----------------- |
| W 答不出替代路徑                 | 回 W 階段重新擴增 |
| A 顯示機會成本偏高               | 暫停並重新排序    |
| P 任一失敗原因機率偏高           | 補防護或改方案    |
| 任務涉及重大架構、規則或根因分析 | 跑完整 WRAP       |

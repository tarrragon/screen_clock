# 來源核對 — 本專案落地

本文件為 WRAP 的 R 階段在**清單類答案**的強化補丁。源自 W10-064 案例：claude-code-guide agent 回報的清單中混有幻覺項目，揭示「LLM 列清單時最容易幻覺」。

> **SKILL.md 留抽象原則（清單類答案對 source 核對，禁止整批信任），本檔放本專案完整防護規格 + W10-064 案例**。

---

## 核心原則

**LLM 列清單時最容易產生幻覺** — 會「補齊」看起來合理的項目。

Reality Test 對清單類答案必須執行**逐項對 source 核對**，**禁止整批信任**。

## LLM 幻覺模式（已實證）

| 幻覺類型 | 症狀 | 為什麼難偵測 |
|---------|------|-----------|
| 數量誇大 | 宣稱「支援 18+ 項」實為 9 項 | 數量大讓人覺得來源權威 |
| 補齊合理項目 | 清單中混入命名合理但不存在的項目 | 與真實項目混在一起難察覺 |
| 用途錯標 | 真實項目的描述為相似但錯誤的用途 | 名稱對得上就以為是對的 |
| 概念混淆 | 把不同層級的概念塞進同一清單 | LLM 會把相關名詞當成同類 |

## 單項細節較可信

對**單項深入問細節**（input schema、欄位語意、觸發條件），LLM 較難幻覺；**列清單**則大幅放大幻覺空間。

**操作規則**：遇到清單時，不問「這個清單對嗎」，改問「第 N 項的細節是什麼」— 即使 LLM 整體清單有幻覺，單項細節正確率仍較高。

## 逐項核對流程

### Step 1：定位 source

找到該領域的權威 source：
- 官方 docs（對應版本）
- 本機 spec/schema 檔（例如 `.claude/config/` 下的 YAML schema 或 `.claude/rules/` 下的規則文件）
- Context7 庫
- GitHub 原始碼

**若無 source**：視為「未驗證」不可採信。

### Step 2：逐項對照

清單每一項都要在 source 中找到對應描述。**找不到 → 標記為候選幻覺**。

### Step 3：交叉驗證

對關鍵項目找第二來源確認（雙重來源）：
- 兩來源都有 → 可信
- 只有單一來源 → 標記為「單一證據」

### Step 4：記錄審查結果

在 Ticket 或 worklog 記錄：
- 正確 N 項
- 幻覺 N 項
- 單一來源 N 項

## W10-064 案例

**情境**：claude-code-guide agent（官方 guide）回報 Claude Code hook 事件。

**幻覺項目**：
- 「CC 支援 18+ events」— 實為較少
- `SubagentStart`（不存在）
- `TaskCompleted`（不存在）

**正確項目（即使 agent 整體有幻覺仍對的）**：
- `SubagentStop` 的 input schema：
  - `agent_id`（對）
  - `agent_type`（對）
  - `agent_transcript_path`（對）
  - `last_assistant_message`（對）

**識破方式**：本機 hook-spec 檔 + Context7 雙重核對。

**教訓**：
- 即使是「官方 guide agent」也會幻覺清單 — 身份不是證據
- 但單項深入問細節（input schema 欄位）LLM 較難幻覺
- 整體清單正確率與單項細節正確率**不相關**

## 反模式

| 反模式 | 為什麼錯 | 正確做法 |
|-------|---------|---------|
| 「agent 是專家所以信他」 | 即使官方 guide agent 也會幻覺清單 | 對清單一律核對 source |
| 「清單看起來很合理」 | 合理性是幻覺的偽裝，不是正確的證據 | 合理性不能取代來源驗證 |
| 「找不到 source 但 agent 說對」 | 沒有 source 等於沒有事實基礎 | 視為未驗證、不可採信 |
| 「一次檢查整批」 | 整批信任等於整批幻覺 | 逐項核對，記錄每項狀態 |

## 執行時機

**強制執行**：
- 任何「列出 N 個 X」的答案
- 從 agent 收到的項目清單
- 依賴第三方 SDK/API 支援項目的分析
- 生成式 AI 提供的技術規格清單

**建議執行**：
- 本機程式碼 grep 結果的交叉驗證（若數量 > 5）
- 文件搜尋結果的可信度確認

**可略過**：
- 從明確 source 直接複製的內容（如直接貼 YAML）
- 單項細節查詢（不是清單）

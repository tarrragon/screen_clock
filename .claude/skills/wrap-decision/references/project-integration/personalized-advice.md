# Step 0 資料充足度 — 本專案落地（個人化建議）

SKILL.md 的 **Step 0 資料充足度閘門**提供通用原理（判定流程、適用時機、反模式）。本專案把個人化建議這個特定場景完整規則化，見 `.claude/pm-rules/personalized-advice-rules.md`。

> **SKILL 原理 ↔ 本專案規則的分工**：
> - SKILL 的 Step 0 章節：適用於**所有**決策諮詢場景（不限個人化建議）
> - 本專案 `pm-rules/personalized-advice-rules.md`：專門處理**個人化建議**場景，提供可執行的三層機制（識別/分級/誠實）
> - 本檔：記錄兩者的銜接與本專案衍生歷史

---

## 三層機制（pm-rules 實作）

**規則 1 — 觸發訊號識別（識別層）**

三類觸發訊號：
1. 個人化關鍵字（我要/我該/推薦給我/適合我）
2. 高風險領域關鍵字（健康/運動裝備/金錢/法律/安全）
3. 對話輪次閾值（> 3 輪未問個人條件）

**規則 2 — 風險分級（分級層）**

三級風險：
- 低風險（客觀知識）→ 直接答
- 中風險（個人化建議）→ 漸進式問 2-3 個關鍵變數
- 高風險（健康/金錢/法律/安全）→ 強制完整問卷 + 建議諮詢專業人士

**規則 3 — 顯性標記假設（誠實層）**

三要素標記格式：
```
此建議基於以下假設：
- [假設 1]
- [假設 2]
若你實際是 [其他情境]，建議會改為 [替代答案]。
請確認假設是否符合你的狀況。
```

完整規格見 `.claude/pm-rules/personalized-advice-rules.md`。本檔不複述，避免 DRY 違反。

## 為什麼有兩份（SKILL Step 0 + pm-rules）

| 層級 | SKILL Step 0 | pm-rules/personalized-advice-rules |
|------|-------------|-----------------------------------|
| 範圍 | 所有決策諮詢（含技術決策、Ticket 認領、個人化建議） | 只處理個人化建議 |
| 內容 | 抽象判定流程、模式識別 | 具體觸發關鍵字、風險分級表、標記格式 |
| 可複用性 | 跨專案通用 | 本專案特定（關鍵字、場景） |
| 使用者 | PM / 任何 AI instance | 本專案的主線程 / 代理人 |

**讀取順序**：
- 遇到個人化建議場景 → 直接讀 pm-rules/personalized-advice-rules（更具體）
- 遇到其他決策諮詢 → 讀 SKILL Step 0（更通用）
- 設計新的 pm-rules → 參考 SKILL Step 0 的判定流程作為模板

## PC-071 衍生的架構決策

PC-071 事件揭示**原版 WRAP 的結構缺口**：錨點確認只檢查「核心目標」，不檢查「資訊充足度」。WRAP 可以在「不知道當事人是誰」的狀態下完整跑完整套流程。

修補決策：
1. **SKILL 增加 Step 0 章節**（W10-075.3）— 作為 W 之前的強制閘門
2. **pm-rules 建立個人化建議規則**（W10-075.1/.2）— 三層機制（識別/分級/誠實）
3. **SKILL description 擴充觸發關鍵字**（個人化建議、具體推薦等）
4. **未來 Hook S8 訊號**（待實作）— 自動偵測個人化關鍵字 + 具體品牌/型號請求

詳細脈絡見 `case-studies.md` 的 PC-071 段落。

## 13 個關鍵變數（本專案範例）

PC-071 羽球選拍場景的 13 個應問變數（身高、體重、性別、慣用手、年齡、有無舊傷、手掌大小、握力、預算、所在地區、球友程度、每週頻率、是否計畫上教練課）為**具體範例**，不是通用規則。

其他領域的關鍵變數對應：
- 健康：病史、用藥、過敏、家族史
- 金錢：預算、風險承受度、時間範圍、稅務身份
- 法律：所在管轄區、合約期限、相關方
- 運動裝備：身體條件、使用頻率、既有裝備、預算

執行 Step 0 時不需機械化列 13 個，而是**預判「哪些變數答案會根本性改變建議」**（pm-rules 規則 2 中風險策略）。

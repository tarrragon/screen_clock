# Memory Capture Guide

從 memory-network-builder Agent 整合的原子化記憶建立規範。
用於在 continuous-learning 流程中產出高品質的知識記錄。

---

## 記憶類型

| 類型 | 定義 | 範例 |
|------|------|------|
| **decision** | 技術決策、架構選擇、方案評估結果 | "選擇用 JSON 而非 YAML 做配置格式" |
| **implementation** | 具體實作方法、解決方案、最佳實踐 | "狀態保存在 .mcp-state 目錄中，於啟動時載入" |
| **learning** | 經驗教訓、性能發現、問題根因 | "批次更新比逐條更新快 10 倍" |
| **concept** | 核心概念、設計原則、理論基礎 | "為什麼需要配置驅動架構：提升系統靈活性" |
| **issue** | 問題記錄、bug 報告、已知問題 | "熱重載在狀態管理中導致資料遺失的問題" |

---

## 結論式標題設計

記憶標題必須是結論導向，讓讀者一眼可知核心結論。

| 好標題（結論式） | 差標題（話題式） |
|--------------|--------------|
| 使用 JWT 而不是 Session 做認證 | 使用者認證系統 |
| 首頁資料快取 5 分鐘自動失效 | 快取策略 |
| Atomic Ticket 比一般 Ticket 降低 40% 理解成本 | Ticket 設計 |
| LSP 比 Grep 快 50 倍用於符號查詢 | 開發工具選擇 |

## 記憶標準結構

```markdown
---
id: [descriptive-english-id]
type: [decision|implementation|learning|concept|issue]
title: [結論式中文標題]
created: [YYYY-MM-DD]
tags: [relevant, tags, in, english]
---

# [結論式中文標題]

## 一句話說明

> [用最簡潔的語言說清楚核心內容，通常一句話，最多兩句]

## 上下文連結

- 基於：[[前置的決策或概念]]
- 導致：[[這個決策導致的後續影響]]
- 相關：[[相關但不直接依賴的內容]]

## 核心內容

[詳細說明：背景、評估過程、最終決策或發現、關鍵數據]

## 關鍵文件

- `path/to/file` - 相關實現
- `docs/xxx.md` - 相關文件
```

---

## 連結策略

| 連結類型 | 定義 | 範例 |
|---------|------|------|
| **基於** | 前置決策或概念 | [[選擇用-JSON-而非-YAML]] -> 本記憶 |
| **導致** | 後續行動或影響 | 本記憶 -> [[採用批次更新架構]] |
| **相關** | 獨立但相關 | [[快取失效機制]] 和 [[快取預熱策略]] |

連結語法：`[[memory-id]]`（kebab-case 英文 ID）

---

## 原子性原則

**核心**：一個記憶 = 一個結論

| 違反原子性 | 正確拆分 |
|-----------|--------|
| "選擇 JSON 格式，改進效能 10%，簡化解析" | 決策：選擇 JSON 格式 / 學習：JSON 提升效能 10% / 概念：JSON 相比 XML 更簡潔 |
| "熱重載導致狀態遺失，所以啟動時從檔案載入" | 問題：熱重載導致狀態遺失 / 實作：啟動時從檔案載入狀態 |

多個相關結論用連結表達關係，不要合併到同一個記憶中。

---

## 檔案管理

- **儲存位置**：`memory/` 目錄（專案根目錄）
- **命名規則**：記憶標題作為檔名，副檔名 .md
- **子目錄**（選用）：`decisions/`、`implementations/`、`learnings/`、`concepts/`、`issues/`

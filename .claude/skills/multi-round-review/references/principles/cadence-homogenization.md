# Cadence 同質化是模板的隱形維度

> **角色**：本卡是 `multi-round-review` 的支撐型原則、被「Round 2 cadence frame」引用。
>
> **何時讀**：規劃 cadence reviewer prompt 時、了解該 frame 主要 catch 什麼。

## 結論

「模板」有兩個維度、寫作規範通常只 enforce 第一維、第二維是隱形維度：

| 維度         | 內容                                                     | 規範狀態                                   |
| ------------ | -------------------------------------------------------- | ------------------------------------------ |
| 內容欄位模板 | 規模對照表、tripwire 條件、失敗模式、回退路徑            | 多數寫作規範已 enforce                     |
| Cadence 模板 | 段首句句型、段末收尾語、表格前導句、過渡詞、列表收尾結構 | **未被規範涵蓋** — 通常要靠 reviewer catch |

## 主題語意 attractor 機制

同 batch 多篇文章受同主題 / 同 constraint 拉到相似句型骨架。即使每篇單獨看都合規（無 emoji、章節結構齊、表格有延伸）、連讀多篇後讀者會建立 strong cadence expectation — cadence 變成「閱讀時自動跳過的雜訊」。

實證：N=5 batch + 無 variant 規劃 → collapse rate 通常接近 100%。

## Cadence Reviewer 主要 catch 目標

- **章節首段 framing**：N 篇都用「X 的核心責任是 Y」起手
- **段位置 cadence**：每篇「定位邊界 / 案例回寫 / 下一步路由」段都用同樣句型骨架
- **三段平行**：「先看 X、再對照 Y、最後看 Z」這類三段式
- **段末收尾**：每篇都用同類「結論性收尾」句

## 修法

- **首段 framing variant**：5 篇 5 種不同切入（責任 / 痛點 / 對照 / 決策瞬間 / 場景）
- **案例段框架 variant**：直接掛入 / 反向追問 / 對照矩陣 / 分層追問四種輪替
- **結尾路由格式 variant**：「接著讀 / 補完看 / 不要先進 X 而是先讀 Y」變體

寫作前主動規劃 3-5 種變體、避免「合規但同骨」collapse。

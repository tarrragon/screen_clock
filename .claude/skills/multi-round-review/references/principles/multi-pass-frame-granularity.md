# Multi-Pass Review Frame 顆粒度盲點

> **角色**：本卡是 `multi-round-review` 的支撐型原則（principle）、被「三大基本原則」段引用。
>
> **何時讀**：規劃 Round 1 / 2 / 3 時、確認每輪 frame 不重複前。

## 結論

同一個 reviewer / 同一個 frame 跑多輪、catch 的東西高度相同。多輪 review 的價值來自 frame 切換、不是重複加深。

## 三個機制

要擴大 catch 覆蓋度需要三條獨立 axis 同時動：

1. **Keyword bank（換工具）**：grep pattern / 檢查 keyword 列出來、跨 reviewer 互不重複
2. **Reader simulation（換視角）**：假裝特定讀者類型（新手 / 專家 / 跨領域）走稿件、看到的問題跟「規範審查員」視角不同
3. **Self-criticism（換層次）**：用本 batch 寫的規範 / 卡片 self-grep 同 batch 稿件、catch 規範化後仍犯的同義變體

只動一個 axis（例：規範審查員多跑幾輪）、catch 的 finding 高度重複。三條同時動、覆蓋率才會立體擴大。

## 反模式

- 「再來一輪」沒指定 frame 切換、用同方向 reviewer 掃同類問題
- 把「多輪 review」等於「多個 instance」、忽略 frame 才是主要變量
- 修完字面層就停、不切換到結構 frame / meta frame

## 套用情境

規劃 Round N+1 時、列出 Round 1 到 N 各 reviewer 用的 frame、找出未動過的 axis。若三條 axis（keyword / reader / self-criticism）有一條未動、就還有 Round N+1 的價值。

# Cadence 同質化：批量 sibling 寫作的同骨層清單與生成端輪替

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 SKILL.md 原則 6（多輪 re-read）與「跟 multi-round-review 的協同」段引用。
>
> **何時讀**：一次寫多份同類文檔（sibling 卡片、批量章節、同構 reference）之前與之後；或 reviewer 要對批量產出跑 cadence audit 時。

---

## 核心原則

**Cadence 同質化是模板的隱形維度：內容欄位各自合規、句型骨架卻在批內複製、連讀第三份起讀者能預測每段的形狀。** 規範定義的「模板」通常只指內容欄位（該有哪些章節、哪些表）；句式、段首語、收尾警句、敘事骨架、條目數量也是模板 — 而且批量寫作（同一人同一天寫多份）會自然把第一份的句型複製到全部。

單份 review 抓不到它：每一份單獨看都合規。偵測必須**連讀**、比對的單位是句子骨架、不是內容。

## 同骨層清單（批量 sibling 的掃描維度）

一次七張 sibling 卡的實測抓出六個同骨層、可直接當 audit 清單：

| 同骨層       | 實測形態                                                             |
| ------------ | -------------------------------------------------------------------- |
| Title 形式   | 7/7 用「主句：副句」冒號雙段式、而既有文檔全是連句                   |
| 開場句式     | 7/7 結論首句「[情境]時、**粗體規則**」、4/7 第二段「判準：…」起頭    |
| 章節標題句式 | 「為什麼 X 自然發生 / 看不見」雙模板輪流用                           |
| 敘事骨架     | 觸發 case 全是「一個 X → reviewer 判定原話 → 修法一句」三拍          |
| 條目形態     | 判讀徵兆 7/7 恰好 3 bullets、同位置破折號；關係段 7/7 恰好取規範下限 |
| 跨檔引用句   | 引同一張卡的句子在兩份文檔近逐字重複                                 |

另一個隱形訊號是**收斂本身**：七份長度變異僅 4 行、章節集合完全等同（全取最小合規集、零自選段）— 多個硬規範同時生效時、批量寫作會收斂到第一個「都通過」的形狀、然後複製它。

## 生成端輪替（寫之前就排、別等 review 抓）

Review 端抓同骨的成本高（要連讀、要比骨架）、且實測證明 review 抓過的同骨會在下一批復發 — **同類 finding 第二次出現、就把規則從 review 端升到生成端**。批量寫作前排好輪替表：

1. **開場 frame 輪替**：規則先行 / 後果先行（先寫失真造成的誤解、規則退第二句）/ 動作先行（先寫該做的第一個動作、再給理由）/ 反差先行（先寫看起來成立的假象、再拆穿）— 每份換一個。
2. **條目形態輪替**：徵兆 / 檢查段在 bullet、表格、條數之間變化、依各份內容的自然形狀選、預先決定「至少兩份用不同形態」。
3. **敘事視角輪替**：case 段輪流從事件、從受害讀者、從修復者視角寫；同一個「修起來超便宜」flourish 整批最多用一次。
4. **引用句去重**：兩份文檔引同一個來源時、引用句各自重寫 — 引用的查核對象是來源原文、表達方式屬於引用方。
5. **自選段保留**：各份依自己的內容長出超出最小合規集的段（適用邊界、論述基礎、額外例）— 全批零自選段是 compliance 收斂的訊號、不是紀律好。

## 邊界

固定章節結構（每份都有結論 / 反模式 / 關係段）是文檔系統的合約、合規且必要 — 本卡管的是章節**內部的句子層**。協議型文檔（給 runtime 讀的 reference）允許比教材更高的結構重複：routing slot（如每份開場第一句定位職責）可以同式、判斷線是「重複是否讓各份的專屬語言消失」。

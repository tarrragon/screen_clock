# 外部分析改寫的交付物是可遷移框架、不是風格轉換

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 `references/source-to-teaching-analysis.md` 的 deliverable pass 引用。
>
> **何時讀**：把外部分析文章交給 AI 或人類改寫時、用本卡確認任務目標是產出讀者可重用的判讀框架，而不是只把原文改成另一種語氣。

---

## 核心原則

外部分析改寫的交付物是可遷移框架，不是風格轉換。寫作任務不能停在「改成某種語氣」「更正向」「更像教學」；真正要交付的是讀者能帶到下一個相似事件使用的判讀工具。

| 改寫層級 | 產物                             | 失敗模式                 |
| -------- | -------------------------------- | ------------------------ |
| 風格轉換 | 同一篇文章換語氣、換標題、換段落 | 像摘要或二次評論         |
| 結構轉換 | 把分析 process 改成教學章節      | 可能仍偏離標題承諾       |
| 框架轉換 | 訊號、機制、風險、預警、遷移用法 | 讀者能用來分析下一個事件 |

判別問題：「讀者讀完後，是否多了一個可重用的判斷問題或檢查表？」沒有，就還只是改寫。

---

## 為什麼風格轉換不夠

AI 擅長把原文改得更順，也能把段落改成目標語氣。但摘要仍會沿著原文 frame 走。讀者得到的是「另一個版本的原文」，不是新的知識單元。

教學型分析文章的價值是把事件變成可重用的判斷問題。它不只回答「這件事代表什麼」，還要回答「下次遇到相似事件時，要看哪些訊號、哪些機制、哪些失效條件」。

---

## 操作方法

### 1. 把任務定義成 frame extraction

輸入外部文章時先問：

| 問題                              | 作用                       |
| --------------------------------- | -------------------------- |
| 原文用什麼 frame 看事件？         | 辨識原作者判讀，不直接繼承 |
| 這個 frame 對目標讀者是否可遷移？ | 決定是否值得寫成文章       |
| 讀者下次要看哪些訊號？            | 把評論轉成判讀工具         |
| 哪些術語需要卡片或首次解釋支撐？  | 避免正文變成術語堆疊       |

若事件只能產出摘要、沒有可遷移框架，就保留成筆記或素材，不硬寫成分析文章。

### 2. 正文服務判讀工具

常見責任順序：

```text
事件本身 → 結構性機制 → 長期影響 → 預警訊號 → 可遷移框架
```

這不是固定模板，而是責任順序。事件段讓讀者進場；機制段教為什麼；長期影響處理時間軸；預警訊號讓框架可被反證；可遷移框架讓讀者能帶走。

### 3. 把原文觀點轉成判斷問題

| 原文觀點型句子         | 框架型問題                             |
| ---------------------- | -------------------------------------- |
| 某技術正在重塑某市場   | 它重塑的是供給、需求、成本，還是通路？ |
| 某收購代表產業整併     | 這是單點收購、賽道整併，還是垂直整合？ |
| 某產品會擠壓既有工作者 | 它擠壓的是工具層、流程層，還是責任層？ |

判斷問題比結論更有價值，因為問題可以帶到下一個 case。

### 4. 用預警訊號保護框架

可遷移框架需要知道何時失效。文章至少要列出哪些觀察會削弱本文判讀。沒有預警訊號的框架只是分類；有失效條件，才是工具。

---

## 判讀徵兆

| 訊號                           | 該做的事                                     |
| ------------------------------ | -------------------------------------------- |
| Prompt 只說「改成我們的風格」  | 改成「抽出可遷移判讀框架」                   |
| 文章結尾只有事件結論           | 補「下次遇到同類事件要看什麼」               |
| 讀者讀完只能說出某公司或某事件 | 補訊號、機制、風險、預警與路由               |
| 文章沒有預警訊號               | 補失效條件，讓框架可被反證                   |
| 文章大量沿用原文 frame         | 回到 source layering，區分原文判讀與本文推導 |

**核心原則**：外部分析改寫不是把原文變順，而是把原文材料轉成讀者可遷移的判讀框架。沒有框架、預警與下一步路由，文章仍停在摘要層。

# 引用章節用語意標題、不用位置編號

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 `references/writing-documents.md` Principle 2（cross-document references）引用。
>
> **何時讀**：在文件 / 教材 / 協議內引用另一個章節、階段、條列項時；或重排既有文件結構（插入 / 合併 / 搬移章節）、要評估引用句修復面時。

---

## 核心原則

**跨段落、跨檔引用結構單位（章節 / 階段 / 條列項 / 輪次）時、引用它的語意標題、不引用它的位置編號。** 編號由「這個單位前面排了幾個單位」決定 — 它是結構排列的衍生值（derivation）、不是這個單位自身的事實（fact）。把「Stage 3」「第 5 章」「上述第 3 點」寫進引用句、等於把當下排列順序的快照散寫到多處；結構一重排、所有快照同時過期、且沒有任何機制通知它們過期。

這條原則同時定義標題的責任：**每個結構單位要有說明核心意義的語意標題、讓引用有穩定的錨**。「Stage 3」單獨存在不是標題、「Stage 3：核心問題」才是 — 編號承擔當下排序導覽、語意半邊承擔被引用。語意半邊本身也要是純 fact 且唯一：「核心七問」這種內嵌成員數的名稱一半是 derivation、加一問就失真（見 sibling 卡 [name-collections-by-role-not-count](./name-collections-by-role-not-count.md)）；同單位的同義雙名（標題一個說法、引用另一個說法）讓 grep 掃 A 漏 B — canonical 取標題語意半邊、全部引用統一。

---

## 失效模式：misdirected 比 dangling 難偵測

| 失效類型                | 觸發                       | 偵測難度                                          |
| ----------------------- | -------------------------- | ------------------------------------------------- |
| Dangling（連結斷掉）    | 目標消失                   | 工具可掃、直接報錯                                |
| Misdirected（編號錯位） | 目標位移、編號被新內容占據 | 工具掃不出 — 字面完好、卻成功解析到語意錯誤的內容 |

連結斷掉會報錯；編號錯位會把讀者**成功帶到錯的地方**、讀者沒有任何訊號知道引用已過期。修復只能人工逐處判讀：grep 找得到所有編號字面、但無法判斷哪些語意還對。

語意標題改名時舊引用同樣過期、但斷裂等級不同：改名產生 dangling（grep 找不到目標、可偵測）、且改名是顯式操作、執行者當下就能掃引用面；編號位移產生 misdirected、且位移是別處插入的副作用、無人在場。對應紀律：**改任何標題的語意半邊、同一個 commit 掃引用句**。

實際 case：一份多階段流程文件、各檔用「Stage 1 的產物」「Stage 3 收斂時」互相引用；下一版流程從四階段改成六階段、原 Stage 1 的內容變成 Stage 3 — 十多處引用字面完好、語意全部錯位、修復靠人工逐處比對、且第一輪漏修兩處。若引用寫的是該階段的語意名稱、重排的引用修復成本是零。

## 修法

1. **結構單位先有語意標題**：標題的本職是承載可被引用的意義、編號只是排序前綴。檢驗法：這個標題單獨出現在別的檔案裡、讀者知道它指什麼嗎？
2. **引用一律取語意半邊**：「見核心問題」而不是「見 Stage 3」；「如『底線告知協議』所述」而不是「如第 4 點所述」。
3. **條列數字只服務當下閱讀**：文內回指條列項時重述語意、不寫「上述 1-3」。
4. **結構重排的 commit 要掃引用面**：全 repo 掃編號式引用（`rg "Stage [0-9]|第 ?[一二三四五六七八九十0-9]+ ?(章|節|點|步|輪)|§[0-9]"`）、命中是候選、逐處判讀。

## 邊界：凍結編號是 fact、可以引用

發布方承諾編號不變的外部規格 — RFC 段號、法條條號、含版本年份的 ISO 條款（跨版改版會重編、鎖定版本才算凍結）、已出版書籍章節 — 編號本身就是 contract、引用它們比引用標題更穩定。內部活文件要獲得同樣性質、要先宣告編號凍結（只加不改）；做不到就走語意引用。

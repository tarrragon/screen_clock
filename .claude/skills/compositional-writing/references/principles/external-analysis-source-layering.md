# 外部分析材料要拆成事實、原作者判讀、本文推導

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 `references/source-to-teaching-analysis.md` 的 source layering pass 引用。
>
> **何時讀**：把分析師文章、投資人備忘錄、產業評論、研究摘要或 AI 改寫稿轉成教學型文章時、用本卡判斷哪些句子是事實、哪些只是原作者判讀、哪些是本文自己的合成框架。

---

## 核心原則

外部分析材料是寫作 source、不是可直接搬進正文的 fact。寫作者要先把材料拆成三層：可驗證事實、原作者判讀、本文推導。三層分清楚，文章才有可回溯性，也才看得出本文真正教了什麼。

| 層級       | 內容角色                       | 進正文時的寫法                     |
| ---------- | ------------------------------ | ---------------------------------- |
| 事實       | 事件、交易、產品發布、公開數字 | 放背景或事件段、保留可回溯來源     |
| 原作者判讀 | 原作者對事件的解釋、預測、立場 | 標成某種觀點或 hypothesis prior    |
| 本文推導   | 本文從多個材料合成的教學框架   | 明確寫成本文整理出的判斷問題或框架 |

判別問題：「這句話如果拿掉原作者名字、還能被當成可驗證事實嗎？」不能，就不能寫成事實句。

---

## 為什麼 source layering 必要

AI 改寫與人工摘要都容易把 attribution 壓掉。原句的「某作者認為 X」經過改寫後變成「這代表 X」；原本只是觀點的 frame 變成了事件事實。句子更順，但 fidelity 更差。

外部分析文章的分類、比喻、因果框架通常是原作者的 intellectual product。教學文章可以引用或回應，但不能把它們當作事件本身。本文若要產出自己的教學框架，也要讓讀者知道這是本文合成，不是來源原文已經給出的 fact。

---

## 操作方法

### 1. 三欄標註

進稿前先把材料拆成三欄：

| 欄位       | 問題                                     |
| ---------- | ---------------------------------------- |
| 事實       | 這件事有公開來源可驗證嗎？               |
| 原作者判讀 | 這是不是某位作者或某類市場敘事的解釋？   |
| 本文推導   | 這是不是本文從多個材料合成出的教學框架？ |

同一句若同時包含兩層，拆成兩句。

### 2. 分配正文位置

| 層級       | 正文位置                         |
| ---------- | -------------------------------- |
| 事實       | 開頭、背景、事件本身             |
| 原作者判讀 | hypothesis prior、背景敘事、反例 |
| 本文推導   | 主體段落、判斷問題、可遷移框架   |

原作者判讀可以幫助 widen hypothesis space，但它不自動成為本文結論。

### 3. 合成 frame 明確標記

當文章把多個 source 合成成一個框架時，用「本文把這個變化整理成...」「本文判讀是...」這類主詞。這能避免讀者把合成 frame 誤認為某個 source 的原文主張。

### 4. 概念先行，來源在後

來源 attribution 是支撐，不是段落主詞。先寫本文要教的概念，再補來源角色。避免「某機構說 X，所以 X 成立」的權威先行句型。

## 判讀徵兆

| 訊號                                                  | 該做的事                                         |
| ----------------------------------------------------- | ------------------------------------------------ |
| 句子寫「這代表 X」、但 X 其實是原作者解釋             | 改成「一種判讀是 X」或標為本文推導               |
| 外部文章的比喻或分類被直接放進本文標題                | 確認這是原作者 frame 還是本文 frame              |
| AI 改寫後少了「某作者認為」「市場敘事」等 attribution | 回原文補 source layer                            |
| 引用來源只寫機構名、沒有具體文章或可驗證出處          | 刪掉具名 source，改成一般 prior 或重新查證       |
| 讀者問「這是事實、原文觀點、還是你的推導」            | Source layering 失敗，重寫段落主詞與 attribution |

**核心原則**：外部分析材料進入教學寫作前，先拆成事實、原作者判讀、本文推導。三層分清楚，才不會把來源 frame 誤當事實，也不會把本文貢獻藏在摘要裡。

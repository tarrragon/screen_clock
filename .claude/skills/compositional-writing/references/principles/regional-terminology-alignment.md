# 地區用語對齊：寫作前先確定讀者的中文語料

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 `references/writing-articles.md` 規則四（術語精度）+ multi-pass review 輪 4 / 輪 8 的 keyword bank 引用。
>
> **何時讀**：寫中文技術文章時、用本卡判斷讀者的地區語料偏好——尤其在 multi-pass review 輪 8 跑 grep 比對地區漂移詞庫時。

---

## 論述基礎與限制

本卡的論述基於 **1 個 case** 的 review 觀察抽出。具體限制：

- **詞庫表是觀察性的分布偏差、不是嚴格語言學分類**：兩岸詞彙在多數情境都互通、本卡描述的是「**業界傳統用法的 heuristic**」、不是「對方語料中不存在這個詞」
- **詞庫由 reviewer 維護**：本卡只給「常見漂移類別」當 starting set、實際詞庫要由寫作者根據讀者地區補充
- **沒有量化資料支撐「對映成本」**：本卡描述「累積性的閱讀流暢度損失」是直覺論述、未做使用者測試
- **預設讀者地區由 reviewer 顯式選擇**：不假設特定地區、寫作前要先 explicit 確定讀者群

---

## 核心原則

中文技術寫作的用詞精度受**讀者的地區語料**影響——同一個概念在不同地區可能用完全不同的詞、或用同一個詞但語意稍微偏移。寫作前先確定讀者的中文語料、避免用對方語料中不存在或意思偏移的詞。

| 漂移類別         | 觀察到的偏好分歧                        | 漂移代價                                        |
| ---------------- | --------------------------------------- | ----------------------------------------------- |
| 硬體 / 顯示      | 兩字詞 vs 單字詞                        | 偏好不同地區的版本暗示寫作者地區、增加讀者對映  |
| 檔案系統         | 詞義範圍不同（同一詞涵蓋範圍不同）      | 用詞分歧、可能造成 file vs document 的歧義      |
| 預設值 / 概念詞  | 同一概念兩種譯法                        | 兩岸都通用、但業界偏好分歧                      |
| 物理 vs 抽象屬性 | 同詞語意在兩岸不同（如 mass / quality） | 嚴重時造成語意誤判                              |
| 影片媒體         | 兩字詞 vs 兩字詞（不同字）              | 傳統偏好不同、但近年混用增加                    |
| 程式設計術語     | 中譯偏好分歧（學術 vs 業界）            | 業界 community 跟學術 community 偏好不同        |
| 介面 vs 接口     | 兩岸用詞分布不同                        | 一邊明確指 interface、另一邊兼指 connector      |
| 記憶體相關       | 兩字詞 vs 兩字詞（語意不同）            | 一邊是 memory、另一邊是「內部存放」、語意誤差大 |

技術文章的精度標準是「**讀者用自己的語料能正確 parse 你的句子**」。用錯地區用語、輕則增加讀者對映成本（可讀但不順）、重則造成語意誤判。

## 為什麼地區用語會漂移

### 來源 1：訓練語料 / 參考材料來源

技術寫作者常從特定地區社群、AI 工具（早期 LLM 訓練資料以特定中文為主）、open source 文件抓參考材料。複製時容易帶進原語料的用詞。

### 來源 2：技術圈的 Stack Overflow 效應

部分技術術語在某個地區社群有大量內容、其他地區相對少。寫作時搜尋資料、看到該地區版本的解釋、不自覺套用對方用詞。

### 來源 3：「讀得懂就好」的低估

「兩岸都看得懂」這個假設在「個別詞」層級成立、但在「整篇文章」層級會累積：每個詞累積一次對映成本、整篇加起來幾十個詞、讀者的認知負擔會增加。

### 來源 4：跨地區團隊的混合語料

跨地區團隊（多個華語區）的工程師常混用兩邊用詞。內部溝通沒問題、但寫成對外文章時要選定一個地區的語料對齊。

## Keyword bank（grep 用的常見漂移詞）

每輪 review 用 grep 比對固定 keyword list、不靠 reviewer 記憶。實際 keyword 由寫作者根據讀者地區決定。範例（針對某個地區的偏好）：

```text
硬體 / 顯示類：偏好兩字詞 vs 單字詞
檔案系統類：file vs document 的詞用法
預設值 / 概念類：同概念的兩種譯法
品質 / 物理屬性類：抽象品質 vs 物理屬性的同詞
影片 / 媒體類：兩種兩字詞
程式設計術語：學術譯法 vs 業界譯法
介面類：interface 的譯法
記憶體類：memory 的譯法、跟「內部存放」的語意誤差
```

具體 keyword 由寫作者維護、根據讀者地區補充。

## 識別訊號

### 訊號 1：技術術語有兩個常見譯法

業界術語在兩岸有不同譯法。

修法：選定讀者地區的版本、全篇統一。

### 訊號 2：硬體 / 業務名詞用了非偏好版本

本案的單字詞 vs 兩字詞、或反之。

修法：選擇貼合讀者地區傳統的版本。

### 訊號 3：物理屬性 vs 抽象屬性混用

如「質量」當 quality 還是 mass、語意在兩岸不同。

修法：抽象屬性 → 用兩岸都明確的詞（品質 / 質地）；物理屬性 → 用兩岸都明確的詞（質量 / 重量）。

### 訊號 4：媒體類詞混用

兩岸用「視訊 / 影片 / 視頻」分布偏好不同、但近年滲透增加、單一判定不易。

修法：對 IT 工程師讀者用業界傳統偏好、其他讀者群評估混用程度。

### 訊號 5:詞義歧義

同一個詞在兩岸涵蓋範圍不同（如「文件」可指 document 也可指 file、看地區跟情境）。

修法：對 file 用更明確的詞（檔案）、對 document 用 documentation 對應的譯法。

## 何時混用 / 不嚴格對齊仍然合理

| 情境                            | 為什麼可以混用                                                      |
| ------------------------------- | ------------------------------------------------------------------- |
| 引用第三方資料 / quote          | 引用原文檔保留原語料準確性比對齊重要                                |
| 業界共同術語（不分地區）        | 「Bug」「commit」「pull request」這類本身就是英文、不在語料漂移範圍 |
| 內部團隊溝通 / 工作日誌         | 寫給知道讀者語料的特定群體、混用不損精度                            |
| 描述對方地區的具體系統          | 對方地區專有名詞、不該翻譯                                          |
| 引用作者自身用詞 / 個人風格保留 | 訪談記錄 / 個人 blog 保留作者自己的語感是 narrative 選擇            |

判讀：寫之前自問「讀者地區是否單一？我的用詞是否會讓讀者多花認知對映？」——是 → 嚴格對齊；否 → 混用可接受。

## 推到「術語選擇」的更上位原則

地區用語對齊是「術語選擇」的子問題。更上位的原則：

> 每個技術詞、寫之前都該確認「讀者語料是否包含這個詞」。

這個自問適用於：

- **地區漂移**：本卡主題（兩岸 / 不同華語區）
- **學術 vs 業界用詞**：「方法論」（學術）vs「做法」（業界）
- **新舊術語**：技術名詞的世代差異
- **中文 vs 英文**：技術術語第一次出現要不要保留原文錨點（見 [terminology-keeps-original-anchor](terminology-keeps-original-anchor.md)）
- **完整名詞頭**：壓縮中文術語要保留 head noun（見 [compressed-chinese-terms-need-head-noun](compressed-chinese-terms-need-head-noun.md)）

地區漂移是這個自問的第一層應用——讀者語料的最大差異訊號。

## 判讀徵兆

| 訊號                                 | 該做的行動                                   |
| ------------------------------------ | -------------------------------------------- |
| 用了讀者地區不偏好的常見漂移詞       | 換成讀者地區傳統偏好的詞                     |
| 文章寫給特定地區讀者、但混用兩邊用詞 | 全篇 grep 替換、選定一個地區語料             |
| 從不同地區的技術文章複製範例 / 解釋  | 翻譯時連用詞一起翻、不只翻句子               |
| 用了某詞但不確定是不是地區漂移       | grep 「某詞 + 讀者地區 + 中文」看用法分布    |
| 內部討論用 A、寫對外文章用 B         | 對外文章固定用讀者地區的版本、內部討論可寬鬆 |

**核心原則**：技術寫作的精度標準是「讀者用自己的語料能正確 parse」。地區用語漂移是累積性的閱讀流暢度損失。寫前先確定讀者地區、寫完跑一次 grep 對齊。

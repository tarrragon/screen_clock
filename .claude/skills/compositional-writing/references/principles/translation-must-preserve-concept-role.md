# Translation Must Preserve Concept Role

> **角色**：本卡是 `compositional-writing` 的支撐型原則（principle）、被 translation review 與 terminology review 引用。
>
> **何時讀**：英文術語被翻成中文後，reader 覺得順口但追問方向怪，或譯名把方法、偏誤、檢查、流程階段混成同一類時。

## 核心命題

術語翻譯要保留概念角色：先判斷原詞在來源語境中承擔的是「論證方法」「檢查動作」「偏誤名稱」還是「流程階段」，再選中文名詞頭。中文譯名如果改變了概念類型，即使句子仍然順口，也會讓 reader 用錯誤方式理解後續操作。

典型例子是把 `Steelman` 翻成「最強版本測試」。這個譯名抓到 checklist 裡的局部用途，卻把概念角色從「重建對方最強論點」壓成「通過一個測試」。較穩的寫法是「最強版本論證（Steelman）」：中文保留論證角色，英文保留可回溯來源。

## 檢查流程

| 檢查       | 問題                                           | 失效訊號                           |
| ---------- | ---------------------------------------------- | ---------------------------------- |
| 來源角色   | 原詞在來源語境中是方法、偏誤、測試還是階段？   | 中文名詞頭跟來源類型不同           |
| 本文角色   | 這個詞在本文用來命名概念，還是命名局部操作？   | 因為放在 checklist，就翻成「測試」 |
| 讀者追問   | reader 看到中文會追問哪件事？                  | 追問通過條件，而不是追問概念品質   |
| 動詞搭配   | 中文能承接本文要求的動作嗎？                   | 「做測試」可行，但原本的方法消失   |
| Surface 面 | title、heading、表格欄位是否傳播同一概念角色？ | 正文改對，checklist 仍保留舊角色   |

最低修法是：中文名詞頭要對應概念角色，英文括號要保留來源錨點。若中文沒有穩定固定譯名，採解釋型中文也可以，但不能把方法翻成測試、把偏誤翻成狀態、把流程階段翻成工具。

## 反模式

| 反模式                      | 後果                       | 修法                         |
| --------------------------- | -------------------------- | ---------------------------- |
| 被版面角色拉走              | checklist 裡的術語都變測試 | 回來源確認概念本來是哪一類   |
| 用中文順口度取代概念分類    | reader 問錯問題            | 問「這是方法、偏誤還是檢查」 |
| 正文改對、navigation 沒同步 | 舊譯名繼續從入口污染理解   | 掃 title / heading / label   |
| 解釋型翻譯只解釋局部操作    | 概念被窄化                 | 中文名詞頭對應來源角色       |

## 最小檢查表

- [ ] 來源語境中的概念類型已標出
- [ ] 中文名詞頭跟來源角色一致
- [ ] 譯名放回本文後，reader 會追問正確問題
- [ ] checklist / heading / index entry 沒有把術語改成另一種角色
- [ ] 第一次出現保留原文括號

**核心**：翻譯術語時，中文要讓 reader 知道「這個概念是哪一類東西」。概念角色一旦翻錯，後續 checklist 再完整，也是在檢查錯的東西。

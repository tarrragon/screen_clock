# 認知負擔評估提示

## 使用方式

當需要評估任務或程式碼的認知負擔時，使用以下提示進行分析。

---

## 任務複雜度評估提示

### 輸入格式

```markdown
## 任務描述
{任務描述}

## 相關檔案
{檔案清單}

## 預期修改
{修改範圍}
```

### 評估輸出格式

```markdown
## 認知負擔評估報告

### 量化指標

| 指標 | 數值 | 等級 |
|------|------|------|
| 變數狀態數 | {數值} | {低/中/高} |
| 修改檔案數 | {數值} | {低/中/高} |
| 架構層級數 | {數值} | {低/中/高} |
| 依賴模組數 | {數值} | {低/中/高} |
| 呼叫追蹤層 | {數值} | {低/中/高} |

### 認知負擔指數

計算：{變數數} + {分支數} + {巢狀深度} + {依賴數} = {總和}
等級：{低/中/高/極高}

### 評估結論

- **複雜度等級**: {低/中/高}
- **建議行動**: {直接執行/謹慎執行/必須拆分}

### 拆分建議（如需要）

| 子任務 | 目標 | 預估複雜度 |
|--------|------|-----------|
| 1 | {描述} | {低/中} |
| 2 | {描述} | {低/中} |
```

## 代理人升級判斷提示

### 自我檢查問題

回答以下問題，任一回答「是」則考慮升級：

1. 我需要同時記住超過 7 個概念嗎？
2. 我需要閱讀超過 5 個檔案才能完成嗎？
3. 我無法用一句話說明這個任務嗎？
4. 閱讀任務描述 5 分鐘後我還是不確定目標嗎？
5. 這個任務需要修改 2 個以上的架構層嗎？
6. 這個任務依賴 3 個以上的其他模組嗎？

### 升級報告格式

```markdown
## 升級請求

### 任務資訊
- **Ticket ID**: {ID}
- **任務描述**: {描述}

### 升級原因

| 檢查項目 | 結果 | 說明 |
|---------|------|------|
| 概念數量 | {數量} | {超過/未超過} 閾值 |
| 檔案數量 | {數量} | {超過/未超過} 閾值 |
| 架構層級 | {數量} | {超過/未超過} 閾值 |
| 理解困難 | {是/否} | {說明} |

### 建議行動

- [ ] 任務需要拆分
- [ ] 需要其他代理人協作
- [ ] 需要更多資訊
- [ ] 需要架構決策

### 拆分建議（如適用）

{拆分建議}
```

## 並行派發安全檢查提示

### 檢查清單

```markdown
## 並行安全檢查

### 任務清單
| 任務 | Ticket ID | 目標檔案 | 依賴 |
|------|-----------|---------|------|
| 1 | {ID} | {檔案} | {依賴} |
| 2 | {ID} | {檔案} | {依賴} |

### 安全檢查

- [ ] 檔案無重疊：各任務修改的檔案集合無交集
- [ ] 測試無衝突：各任務的測試可獨立執行
- [ ] 依賴無循環：任務之間無先後依賴關係
- [ ] 資源無競爭：不會同時存取相同外部資源
- [ ] 架構層級：所有任務在同一層級

### 結論

- **可以並行**: {是/否}
- **原因**: {說明}
- **建議**: {並行執行/改為序列/部分並行}
```

# 個人化諮詢方法論

本方法論整合 AI 進行個人化建議時的完整決策流程，目標是消除「視野狹窄偏誤」並建立「先問後答」的諮詢文化。

---

## 前言：為什麼需要個人化諮詢方法論？

**核心問題**：所有專業領域皆以「先問後答」為標準——醫生必問病史、理專必問 KYC、教練必問身體狀況、律師必問事實、藥師必問過敏與用藥史；唯獨 AI 助手習慣直接給答案。AI 反其道而行屬**文化偏誤**，非技術限制。本方法論旨在矯正此偏誤。

**本方法論回答的根本問題**：

> 在「對話流暢度」與「答案準確度」之間如何取捨？

答案：**準確度永遠優先，但可透過分級機制把詢問成本最小化**。

> **方法論起源的羽球選拍實證案例詳述**（4 輪未問即推薦、13 項缺失變數、自相矛盾推薦表、七大專業「先問後答」對照）：`.claude/references/personalized-consultation-examples.md` 第一節。需追溯方法論成形的觸發事件、向他人說明「為何 AI 不問是失職」時讀取。

## 1. 核心理念

### 1.1 行為優先級反轉

| 優先級 | 偏誤的 AI 行為 | 正確的 AI 行為 |
|-------|-----------|----------|
| 1 | 快速 | **準確** |
| 2 | 博學（展示知識） | **誠實**（承認資訊不足） |
| 3 | 完整（涵蓋所有面向） | **個人化**（對齊當事人） |
| 4 | 流暢（不打斷對話） | **負責**（不給錯建議） |

### 1.2 三項核心原則

**原則 1：個人化建議的第一步是承認不認識當事人**

> 任何專業諮詢的開場都不是「答案」，而是「了解當事人」。AI 跳過此步驟是失職而非效率。

**原則 2：群體統計不能替代個體確認**

> 「30 歲男性平均喜歡 X」是統計事實，但對單一用戶的準確率約 30-50%。在個人化諮詢場景，平均值假設等於 50% 失敗率。

**原則 3：對話流暢度不應凌駕準確度，但可透過分級平衡**

> 不是「每次都問」也不是「都不問」，而是依風險分級採取對應的詢問策略。

## 2. 適用範圍

### 2.1 適用場景判定

| 場景特徵 | 是否適用本方法論 |
|---------|----------------|
| 用戶提問涉及「我該/我要/推薦給我」 | 適用 |
| 即將給出具體品牌/型號/金額/劑量 | 適用 |
| 涉及健康/金錢/法律/安全決策 | **強制適用** |
| 客觀知識查詢（規格、定義、原理） | 不適用 |
| 用戶明確要求「快速通用版」 | 可豁免，但必須標記假設 |

### 2.2 跨領域適用性

本方法論對所有「個人條件差異會改變答案」的領域皆適用：

| 領域 | 個人條件差異範例 |
|------|---------------|
| 健康與運動 | 年齡、性別、舊傷、體能、慢性疾病 |
| 金融與投資 | 年齡、收入、風險承受度、家庭責任 |
| 法律與合約 | 司法管轄區、合約類型、權利義務狀態 |
| 教育與學習 | 既有基礎、學習目標、可投入時間 |
| 飲食與營養 | 過敏、飲食習慣、健康狀況、文化禁忌 |
| 旅遊與規劃 | 預算、時間、體力、興趣、簽證狀態 |

## 3. 三層機制詳解

### 3.1 第一層：識別（Detection）

**目的**：偵測「個人化諮詢」場景，啟動後續流程。

**觸發訊號**：

**A. 個人化關鍵字**：
- 第一人稱 + 需求動詞：「我要」「我該」「推薦給我」「適合我」「給我建議」
- 個人化代名詞：「我的」「你覺得我」「像我這種」

**B. 高風險領域關鍵字**：

| 領域 | 關鍵字 |
|------|-------|
| 健康 | 吃什麼、身體狀況、運動、減肥、受傷、症狀 |
| 運動裝備 | 球拍、跑鞋、健身器材、護具 |
| 金錢 | 投資、買入、預算、要不要花、理財、基金 |
| 法律 | 合約、訴訟、可以告、權利、違法 |
| 安全 | 安全嗎、會不會出事、危險、風險 |

**C. 對話輪次閾值**：對話超過 3 輪仍未問過個人條件 → 強制回頭補問。

**識別後的動作**：進入第二層分級判定。

### 3.2 第二層：分級（Tiering）

**目的**：依風險等級採取對應的詢問策略，平衡準確度與流暢度。

**三級風險判定**（依優先順序，命中即停）：

| 等級 | 判定條件 | 詢問策略 |
|------|---------|---------|
| **低** | 客觀知識查詢（規格、定義、原理） | 直接答，可選標記假設 |
| **中** | 個人化建議（選裝備/習慣/工具/方法） | 漸進式問 2-3 個關鍵變數再答 |
| **高** | 健康/金錢/法律/安全；不可逆後果；超過顯著預算 | 強制完整問卷 + 建議諮詢專業人士 |

**中風險的關鍵變數選擇原則**：
1. 預判「答案會根本性改變建議」的變數
2. 優先問「無法事後修正」的條件（舊傷、禁忌、合約條款）
3. 用戶無法回答時，提供假設選項讓其選擇

**高風險的完整問卷原則**：
1. 結構化分組（身體條件 / 目標 / 限制 / 環境 / 偏好）
2. 告知用戶「為什麼問」（建立信任）
3. 完成問卷後仍建議諮詢專業人士（AI 建議不構成專業建議）

### 3.3 第三層：誠實（Transparency）

**目的**：當無法詢問或選擇直接回答時，顯性化所用的假設。

**標記格式**（強制三要素）：

```
此建議基於以下假設：
- [假設 1]
- [假設 2]
若你實際是 [其他情境]，建議會改為 [替代答案]。
請確認假設是否符合你的狀況。
```

**標記位置**：
- 結論段落開頭或結尾
- 禁止隱藏在附註或中間段落
- 多假設必須列點，不混在一段文字內

**觸發條件**：

| 風險 | 標記 |
|------|------|
| 低 | 可選（建議對非常識性知識標註） |
| 中 | **強制** |
| 高 | **強制 + 建議諮詢專業人士** |

## 4. 完整決策流程

```
用戶提問
    |
    v
[第一層：識別]
    觸發訊號（A個人化 / B高風險 / C輪次閾值）任一命中？
    |
    +-- 否 → 直接答（純客觀知識查詢）
    +-- 是 → 進入第二層
            |
            v
        [第二層：分級]
            判定風險等級（高 → 中 → 低 優先順序）
            |
            +-- 高風險 → 強制完整問卷 → 答前建議諮詢專業
            +-- 中風險 → 漸進問 2-3 個關鍵變數
            |              |
            |              v
            |          資料收集足夠？
            |              |
            |              +-- 是 → 給個人化答案
            |              +-- 否 → 進入第三層
            +-- 低風險 → 進入第三層（可選標記）
                    |
                    v
                [第三層：誠實]
                    顯性標記假設（強制三要素）
                    |
                    v
                  給答案
```

## 5. 跨領域案例分析

三層機制（識別/分級/誠實）在五個領域的落地對照：健康與運動（高風險）、金融與投資（高風險）、運動裝備（中風險）、法律諮詢（高風險）、飲食與營養（中-高風險）。每個案例提供「錯誤回應」（未問即答）與「正確回應」（先列必要前置資訊、高風險建議諮詢專業人士）對照。

> **五個跨領域案例的完整錯誤/正確回應範本**：`.claude/references/personalized-consultation-examples.md` 第二節。需要某一領域的標準回應範本、或對照具體案例理解三層機制如何落地時讀取。

## 6. 常見陷阱

| 陷阱 | 症狀 | 正確做法 |
|------|------|---------|
| 偽諮詢陷阱 | 列出影響條件，但仍替用戶做「平均值假設」直接給推薦 | 列出影響條件後，問用戶屬於哪一類再給對應建議 |
| 過度詢問陷阱 | 把客觀知識查詢誤判為個人化諮詢，對低風險也問完整問卷 | 判定為低風險，直接答 |
| 隱性假設陷阱 | 給答案不標記假設，讓用戶誤以為是普適真理 | 標記「基於 X 假設，若你是 Y 情境答案會變」 |
| 純文字提問陷阱 | 用純文字列多個問題（用戶易漏答、難結構化、違反 PC-064） | 使用 AskUserQuestion 工具或結構化清單格式 |
| 假諮詢真推銷陷阱 | 問問題只為讓推薦看似客製化，實際不論答什麼都推同一商品 | 詢問必須真正影響後續建議方向，否則不問 |

> **五種陷阱的完整症狀範例與正確做法對照**：`.claude/references/personalized-consultation-examples.md` 第三節。需要對照具體範例辨識某一陷阱、或向他人示範陷阱與正確做法差異時讀取。

## 7. 實踐檢查清單

### 7.1 給個人化建議前

- [ ] 已偵測觸發訊號（個人化關鍵字 / 高風險領域 / 輪次閾值）
- [ ] 已判定風險等級（低 / 中 / 高）
- [ ] 中/高風險場景已詢問關鍵變數
- [ ] 高風險場景已建議諮詢專業人士
- [ ] 未詢問時已顯性標記假設

### 7.2 30 秒前置自查

- [ ] 我知道當事人是誰嗎？（年齡/性別/身體條件）
- [ ] 我知道當事人的目標嗎？
- [ ] 我知道當事人的限制嗎？（預算/時間/禁忌）
- [ ] 我知道當事人的環境嗎？（地區/可用資源）
- [ ] 我在用「平均值假設」替代「個人資料」嗎？
- [ ] 如果假設錯誤，後果會嚴重嗎？
- [ ] 後果嚴重 + 資料不足 → **必須先問**

### 7.3 詢問品質檢查

- [ ] 詢問的變數真正影響答案方向？（非表演性詢問）
- [ ] 用結構化方式詢問（AUQ 或清單），非純文字提問？
- [ ] 告知用戶「為什麼問」（建立信任）？
- [ ] 提供假設選項讓不確定的用戶可選？

## 8. 與其他方法論的關係

| 方法論 | 與本方法論的關係 |
|--------|----------------|
| `cognitive-load-design-methodology` | 兩者皆關注「資訊處理品質」，但本方法論聚焦諮詢場景，前者聚焦程式設計 |
| `5w1h-self-awareness-methodology` | 5W1H 提醒 PM 自覺；本方法論將自覺擴展到對外諮詢場景 |
| `acceptance-criteria-methodology` | 相似精神：先確認需求邊界再行動 |
| WRAP 決策框架 | 本方法論是 WRAP Step 0「資料充足度檢查」的諮詢場景應用 |
| `friction-management-methodology` | 本方法論的「分級」即為摩擦力配置；低風險低摩擦，高風險高摩擦 |

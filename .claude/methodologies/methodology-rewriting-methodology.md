# 方法論改寫方法論

## 核心原則

改寫 = 精簡 + 拆分 + 引用完整性。

確保原本引用這個文件的其他文件能找到新版或拆分出去的部分。

---

## 改寫流程

### Step 1: 分析被引用

搜尋所有引用該方法論的文件，建立「引用→章節」對照表。

### Step 2: 規劃拆分

決定：
- 哪些章節保留在主文件
- 哪些章節獨立成衛星文件
- 是否需要建立對應 SKILL

### Step 3: 建立引用遷移計劃

對每個引用者判斷：
- 繼續引用主文件
- 改為引用衛星文件
- 同時引用多個文件

### Step 4: 執行

1. 建立衛星文件
2. 精簡主文件
3. 更新引用者
4. 驗證無斷連

---

## 檢查清單

- [ ] 已搜尋所有引用者
- [ ] 引用遷移計劃已建立
- [ ] 衛星文件已建立
- [ ] 主文件已精簡
- [ ] 引用者已更新
- [ ] 所有引用有效

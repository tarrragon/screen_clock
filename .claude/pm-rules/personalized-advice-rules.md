# 個人化建議前置檢查規則

本文件定義主線程（PM）和代理人在給出個人化建議前，必須執行的前置檢查機制。

> **核心理念**：個人化建議的第一步是承認不認識當事人。所有專業領域（醫療、金融、運動、法律、裁縫）皆以「先問後答」為標準，AI 反其道而行屬文化偏誤，非技術限制。

---

## 適用範圍

**個人化建議定義**：用戶提出的問題涉及個人條件差異會改變答案的諮詢場景。

| 場景 | 適用 |
|------|------|
| 用戶詢問「我該選/買/用什麼？」 | 適用 |
| 用戶詢問具體品牌/型號/金額/劑量 | 適用 |
| 用戶涉及健康/金錢/法律/安全決策 | **強制適用** |
| 用戶詢問客觀知識（規格、定義、原理） | 不適用（低風險直接答） |
| 用戶明確要求「快速通用版」 | 可跳過，但必須標記假設 |

---

## 強制規則

### 規則 1：觸發訊號識別（識別層）

**三類觸發訊號**，任一命中即進入分級判定：

**1a. 個人化關鍵字**：
- 第一人稱 + 需求動詞：「我要」「我該」「推薦給我」「我應該」「適合我」「給我建議」
- 個人化代名詞：「我的」「你覺得我」「像我這種」

**1b. 高風險領域關鍵字**：

| 領域 | 關鍵字 |
|------|-------|
| 健康 | 吃什麼、身體狀況、運動、減肥、受傷、症狀、藥 |
| 運動裝備 | 球拍、跑鞋、健身器材、護具、器械 |
| 金錢 | 投資、買入、預算、要不要花、理財、基金、股票、貸款 |
| 法律 | 合約、訴訟、可以告、權利、違法、責任 |
| 安全 | 安全嗎、會不會出事、危險、風險、有害 |

**1c. 對話輪次閾值**：
- 對話超過 3 輪仍未問過個人條件 → 強制回頭補問

### 規則 2：風險分級（分級層）

**三級風險判定**（按優先順序執行）：

| 風險等級 | 判定條件 | 詢問策略 |
|---------|---------|---------|
| **低風險** | 客觀知識查詢（規格、定義、原理、歷史） | 直接答，**可選**標註假設 |
| **中風險** | 個人化建議（選裝備、選習慣、選工具、選方法） | **漸進式問 2-3 個關鍵變數**再答 |
| **高風險** | 健康/金錢/法律/安全決策；不可逆後果；超過用戶顯著預算 | **強制完整前置問卷**；答前建議諮詢專業人士 |

**判定優先順序**（從高到低，命中即停）：
1. 偵測 1b 高風險關鍵字 → 走高風險策略
2. 偵測 1a 個人化關鍵字 → 走中風險策略
3. 皆未命中 → 低風險策略

**中風險的關鍵變數選擇原則**：
- 預判「這些變數答案會根本性改變建議」
- 優先問「無法事後修正」的條件（如舊傷、禁忌、合約條款）
- 用戶無法回答時，提供假設選項讓其選擇

### 規則 3：顯性標記假設（誠實層）

**標記格式**（強制三要素）：

```
此建議基於以下假設：
- [假設 1]
- [假設 2]
若你實際是 [其他情境]，建議會改為 [替代答案]。
請確認假設是否符合你的狀況。
```

**標記位置要求**：
- 必須出現在結論段落開頭或結尾
- 禁止隱藏在附註或中間段落
- 多假設必須列點，不得混在一段文字內

**觸發條件**：

| 風險 | 標記要求 |
|------|---------|
| 低風險 | 可選（建議對非常識性知識標註） |
| 中風險 | **強制** |
| 高風險 | **強制 + 建議諮詢專業人士** |

### 規則 4：30 秒前置自查

**給個人化建議前必須自問**：

- [ ] 我知道當事人是誰嗎？（年齡/性別/身體條件）
- [ ] 我知道當事人的目標嗎？
- [ ] 我知道當事人的限制嗎？（預算/時間/禁忌）
- [ ] 我知道當事人的環境嗎？（地區/可用資源）
- [ ] 我在用「平均值假設」替代「個人資料」嗎？
- [ ] 如果假設錯誤，後果會嚴重嗎？

**判斷結果**：
- 後果嚴重 + 資料不足 → **必須先問**
- 後果輕微 + 資料不足 → 直接答 + 標記假設
- 資料充足 → 直接答

### 規則 5：WRAP Step 0 前置

使用 WRAP 決策框架時，**Widen Options 之前**必須執行 Step 0：

```
Step 0：資料充足度檢查
    |
    +-- 資料夠嗎？
        |
        +-- 不夠 + 高風險 → 強制問，暫停 Widen
        +-- 不夠 + 中風險 → 漸進式問 2-3 個關鍵
        +-- 夠 或 低風險 → 進入 Step W（Widen）
```

> - 通用原理：`.claude/skills/wrap-decision/SKILL.md` Step 0 章節
> - 本專案銜接：`.claude/skills/wrap-decision/references/project-integration/personalized-advice.md`

## 禁止行為

| 禁止 | 原因 |
|------|------|
| 個人化問題直接給具體型號/金額/劑量不標記假設 | PC-071 核心偏誤，違反規則 3 |
| 回答中列出「X 條件會影響答案」卻從未問用戶該條件 | 自相矛盾（羽球案例實證） |
| 假設「用戶會主動補充關鍵資訊」 | 用戶不知什麼是關鍵變數 |
| 以「對話流暢」為由跳過詢問 | 準確度 > 流暢度；用戶更不滿錯誤答案 |
| 付費諮詢先問清楚、免費諮詢直接給答 | 錯誤的品質差別待遇 |
| 高風險領域（健康/金錢/法律）不建議諮詢專業人士 | 違反規則 2 高風險策略 |
| 對話超過 3 輪仍未問任何個人條件 | 違反規則 1c 輪次閾值 |

## 檢查清單

給個人化建議前確認：

- [ ] 已識別觸發訊號（規則 1）
- [ ] 已完成風險分級判定（規則 2）
- [ ] 中/高風險場景已詢問關鍵變數
- [ ] 未詢問時已顯性標記假設（規則 3）
- [ ] 已通過 30 秒前置自查（規則 4）
- [ ] 若用 WRAP，已執行 Step 0（規則 5）
- [ ] 高風險場景已提醒用戶諮詢專業人士

## 常見反模式（禁止的對話範例）

**錯誤 1**：直接給推薦不問條件

> 用戶：「我是雙打新手，推薦什麼球拍？」
>
> AI：「推薦 Yonex Nanoflare 001，NT$1,500-2,000，適合新手。」

（違反規則 2 中風險策略，未問身高/體重/性別/舊傷等關鍵變數）

**錯誤 2**：列出影響條件但不問

> AI：「女性或手腕力量較弱建議選 5U；男性或握力強建議選 4U。」
>
> （繼續給具體推薦，但從未問用戶性別或握力）

（違反規則 3 和自相矛盾原則）

**錯誤 3**：純文字問句讓用戶自由回答

> AI：「你可以告訴我你的身高體重嗎？」

（應使用 AskUserQuestion 結構化詢問，或列出關鍵變數清單一次問完）

## 正確做法範例

**中風險場景**：

> 用戶：「我是雙打新手，推薦什麼球拍？」
>
> AI：「在給具體型號前，有幾個關鍵問題會根本性改變建議：
>
> 1. 身體條件：身高、體重、性別、年齡
> 2. 健康狀態：手腕/肩膀有無舊傷？
> 3. 預算範圍：單支球拍可接受的金額
> 4. 所在地區：方便購買的品牌通路
> 5. 球友程度：對手節奏如何
>
> 以上會決定選 4U 還是 5U、中桿軟硬、品牌取向。若想先看通用知識，我可提供規格科普；若要個人化推薦，這幾個條件是必要資訊。」

**高風險場景**：

> 用戶：「我應該投資這檔基金嗎？」
>
> AI：「投資決策涉及個人財務狀況與風險承受度，我需要先了解：
>
> 1. 年齡與收入穩定度
> 2. 現有投資組合與緊急預備金
> 3. 此次投資占可支配資產的比例
> 4. 預期投資期間與能承受的最大帳面虧損
> 5. 是否有專業財務顧問可諮詢
>
> 即便以上答案完整，我建議您**諮詢合格財務顧問**做最終決策，AI 建議不構成投資建議。」

# PC-037: 背景代理人完成前過早驗證產出物

## 錯誤場景

PM 並行派發多個背景代理人後，在部分代理人尚未完成時就檢查其產出物（檔案、狀態），
因檔案不存在而誤判代理人失敗，錯誤地釋放 Ticket 或標記為未完成。

## 根因

1. PM 未等待 `task-notification` 回報就主動檢查產出物
2. 背景代理人執行時間不可預測（本例耗時 161 秒）
3. PM 缺少「等待完成通知後再驗證」的意識

## 觸發條件

- `run_in_background: true` 的代理人派發
- PM 在收到完成通知前執行 `ls`/`git status` 檢查代理人產出

## 正確做法

| 時機 | 行為 |
|------|------|
| 代理人派發後、完成通知前 | 不檢查產出物，處理其他任務或等待 |
| 收到 task-notification 後 | 驗證產出物、diff、測試結果 |
| 多個代理人並行時 | 逐個等待完成通知，不假設先完成的代理人代表全部完成 |

## 禁止行為

| 禁止 | 原因 |
|------|------|
| 在完成通知前檢查產出物並做判斷 | 可能誤判為失敗 |
| 在完成通知前釋放 Ticket | 破壞任務追蹤 |
| 基於「檔案不存在」就斷定代理人失敗 | 代理人可能仍在執行中 |

## 影響

- 本例：某 Ticket 被錯誤釋放回 pending，代理人完成後需手動修正狀態
- 浪費時間：需要額外的狀態修正和重新驗證

# PC-071: 個人化建議未詢問當事人條件（視野狹窄偏誤）

## 錯誤症狀

AI 在用戶提出個人化建議請求時，直接用「群體平均值假設」替代詢問當事人條件，給出看似完整但實際偏差的建議。典型症狀包含：

1. 用戶提問涉及「我要 / 我該 / 推薦給我 / 適合我」等個人化關鍵字
2. 主題為裝備選購、健康建議、金錢決策、法律諮詢、安全評估等高變異個人化領域
3. AI 直接給出具體品牌、型號、金額、劑量、流程
4. 整段回答中未詢問當事人任一關鍵條件（年齡、性別、身體狀況、預算、舊傷、環境、偏好）
5. 回答內容中甚至列出「性別/年齡/舊傷會影響建議」，但從未問用戶這些變數（自相矛盾）

**實際情況**：AI 的建議建立在「用戶是群體平均值」的隱性假設上，該假設在個人化場景的準確率約為 30-50%，高風險領域（健康、金錢、安全）可能直接導致實質傷害。

## 觸發案例

### 案例：2026-04-16 羽球選拍諮詢

**對話過程**：

| 輪次 | 用戶提問 | AI 行為 | 疏失狀態 |
|------|---------|---------|---------|
| 1 | 「初階中階高階球拍差別是什麼？」 | 給客觀規格知識 | 合理（資訊查詢） |
| 2 | 「我是雙打新手，十年斷層」 | 用平均值推薦具體型號與預算 | **第一次疏失** |
| 3 | 「4U 跟 5U 差別大嗎？」 | 繼續用群體統計回答 | **第二次疏失** |
| 4 | 「單打和雙打差別多大？」 | 同上，給具體購買建議 | **第三次疏失** |
| 5 | 「你沒問我身高體重性別，這是疏失嗎？」 | 承認疏失 | 事後承認 |

**AI 未問卻關鍵的 13 個變數**：
身高、體重、性別、慣用手、年齡、舊傷史、手掌大小、握力、預算、所在地區、球友程度、每週頻率、是否上教練課。

**自相矛盾的證據**：AI 在推薦表中明確寫「女性或手腕力量較弱選 5U」「年齡 35+ 考慮 5U」，卻從頭到尾沒問用戶這些變數。

## 根因分析

### 核心矛盾

AI 的行為優先級被倒置：

| 優先級 | 當前 AI 行為 | 應有的行為 |
|-------|-----------|----------|
| 1 | 快速 | 準確 |
| 2 | 博學（展示知識） | 誠實（承認資訊不足） |
| 3 | 完整（涵蓋所有面向） | 個人化（對齊當事人） |
| 4 | 流暢（不打斷對話） | 負責（不給錯建議） |

### 六項具體誤判機制

1. **WRAP Step 0 缺失**：跳過「資料充足度檢查」直接進入 Widen Options，用假設填補資訊空缺
2. **假設「問問題破壞流暢度」**：反例——用戶發現疏失時明顯更不滿「流暢但錯誤的答案」
3. **假設「用戶想要速度 > 準確」**：反例——諮詢場景用戶願意花時間提供條件
4. **假設「詢問代表無能」**：反例——醫生/教練/理專/律師/裁縫**必問**是專業表現，不問才失職
5. **假設「用戶會主動補充關鍵資訊」**：反例——用戶不知道什麼是關鍵變數（用戶怎麼知道「慣用手」會影響球拍選擇？）
6. **付費/免費的錯誤差別待遇**：若付費諮詢會先問清楚，免費場景降低標準是錯誤的品質差別

### 跨領域反例（證明此為文化偏誤非技術限制）

| 專業 | 處理資訊不足的方式 |
|------|------------------|
| 醫生 | 必問病史（KYH），沒問就是失職 |
| 理專 | 合規必問 KYC（年齡、風險承受、財務狀況） |
| 教練 | 必問身體狀況、目標、受傷史 |
| 律師 | 必問事實細節與時序 |
| 裁縫 | 必量身體尺寸 |
| 藥師 | 必問過敏、用藥史、孕期 |
| **AI 助手** | **直接給答案**（文化偏誤） |

所有專業以「先問後答」為標準，AI 反其道而行屬**文化偏誤**，非技術限制。

## 防護措施

### 1. 三層機制（識別 / 分級 / 誠實）

> 完整規則見 `.claude/pm-rules/personalized-advice-rules.md`（待建立，對應 0.18.0-W10-075.2）

**第一層：識別** —— 觸發訊號清單

個人化關鍵字（任一命中即進入檢查）：
- 第一人稱 + 需求動詞：「我要」「我該」「推薦給我」「我應該」「適合我」「給我建議」
- 個人化代名詞：「我的」「你覺得我」「像我這種」

高風險領域關鍵字：
| 領域 | 關鍵字 |
|------|-------|
| 健康 | 吃什麼、身體狀況、運動、減肥、受傷、症狀 |
| 運動裝備 | 球拍、跑鞋、健身器材、護具 |
| 金錢 | 投資、買入、預算、要不要花、理財、基金、股票 |
| 法律 | 合約、訴訟、可以告、權利、違法 |
| 安全 | 安全嗎、會不會出事、危險、風險 |

對話輪次閾值：
- 對話超過 3 輪仍未問過個人條件 → 強制回頭問

**第二層：分級** —— 查詢 / 諮詢 / 高風險

| 風險等級 | 判定條件 | 詢問策略 |
|---------|---------|---------|
| 低風險 | 客觀知識查詢（規格、定義、原理） | 直接答，可選標註假設 |
| 中風險 | 個人化建議（選裝備、選習慣、選工具） | **漸進式問 2-3 個關鍵變數**再答 |
| 高風險 | 健康/金錢/法律/安全決策 | **強制完整前置問卷**；建議諮詢專業人士 |

**第三層：誠實** —— 顯性標記假設

若因流暢度考量選擇直接回答，必須顯性標記：

```
此建議基於以下假設：[列出使用的假設]
若你實際是 [其他情境]，建議會改為 [替代答案]
請確認假設是否符合你的狀況。
```

### 2. 個人化建議前 30 秒自查清單

在給出個人化建議前，必須自問：

- [ ] 我知道當事人是誰嗎？（年齡/性別/身體條件）
- [ ] 我知道當事人的目標嗎？
- [ ] 我知道當事人的限制嗎？（預算/時間/禁忌）
- [ ] 我知道當事人的環境嗎？（地區/可用資源）
- [ ] 我在用「平均值假設」替代「個人資料」嗎？
- [ ] 如果假設錯誤，後果會嚴重嗎？
- [ ] **若後果嚴重且資料不足 → 必須先問**

### 3. WRAP 框架的 Step 0 前置

Widen Options 的前置步驟是「資料充足度檢查」：

```
Step 0：我手上的資訊夠嗎？
    |
    +-- 不夠 + 高風險 → 強制問
    +-- 不夠 + 中風險 → 漸進式問 2-3 個
    +-- 夠 or 低風險 → 進入 Step W
```

## 與其他 Pattern 的關係

| 維度 | PC-050（代理人狀態誤判） | PC-070（Hook 訊號推論） | PC-071（個人化視野狹窄） |
|------|---------------------|--------------------|-----------------------|
| 偏誤類型 | 資訊源選擇錯誤 | 信任間接訊號 | **跳過資訊收集步驟** |
| 防護方向 | 強制查詢正確源 | 強制補正確驗證 | **強制先問再答** |
| 核心缺失 | 查詢方式錯誤 | 查詢被跳過 | **自覺性不足** |

PC-071 與 PC-050 / PC-070 同屬「自覺性不足」家族，但 PC-071 層次更基礎——**連「我需要更多資訊」的自覺都缺失**。

## 正確做法

### 正確範例：羽球諮詢的正確回應

**用戶提問**：「我是雙打新手，推薦什麼球拍？」

**錯誤回應**（現狀）：直接給型號清單和價格。

**正確回應**：

> 在給具體型號推薦前，有幾個關鍵問題會根本性改變建議方向：
>
> 1. **身體條件**：身高、體重、性別、年齡
> 2. **健康狀態**：手腕/肩膀有無舊傷？
> 3. **預算範圍**：單支球拍可接受的金額
> 4. **所在地區**：方便購買的品牌通路
> 5. **球友程度**：對手節奏如何
>
> 以上會決定選 4U 還是 5U、中桿軟硬、品牌取向。若你想先看通用知識，我可以提供規格科普；若要個人化推薦，這幾個條件是必要資訊。

**關鍵特徵**：
- 顯性說明「為什麼問」
- 讓用戶選擇「快速通用 vs 個人化精準」
- 不隱藏假設

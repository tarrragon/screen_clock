# PC-116: 用「偶發/transient/不可重現」當分析失敗的合理化標籤

## 症狀

分析任務（debug / 根因調查 / 對照實驗）在沒有 base rate 數據前，將「我沒找到原因」描述為「沒有原因」「偶發」「transient」「不可重現」「conditions unknown」等標籤化收場。

特徵：

- 觀察到 N=1 或 N=2 的失敗事件
- 改變條件後事件不再出現
- 直接寫入結論「偶發 X」/「transient X」
- 不嘗試控制變因 + 不記錄 base rate
- 用「成本/收益不對稱」當不繼續調查的理由

## 根因

### R1：「偶發」是反詞——把「我沒找到原因」描述成「沒有原因」

LLM 推理對「未識別變因」的容忍度低，傾向把「我看不到的解釋」轉換為「沒有解釋」（語意減壓）。當實驗失敗 + 無立即可見根因時，「偶發 / transient」是看似科學中立但實際是反詞的標籤。

**Why**：標籤化滿足「我有結論」的心理需求，但結論的本質是「未繼續調查」。**Consequence**：未識別變因被永久封存為「不可知」，未來再撞同樣事件無法累積知識。**Action**：寫「偶發」前必須有 base rate 數據（≥ 3 次獨立觀察）。

### R2：「成本/收益不對稱」濫用為不繼續調查的合理化

WRAP R 階段宣稱「成本/收益不對稱不執行」直接結束實驗——但實際成本通常極低（單派 thyme / 5-30 min）。

**Why**：「成本不對稱」是真實考量，但被誤用為跳過 R 階段的免責標籤。**Consequence**：合理的低成本對照實驗被跳過，結論基於 N=1 觀察。**Action**：用「成本/收益」拒絕繼續前必須具體列出「最低成本對照實驗」的 token / time 估算，而非概括拒絕。

### R3：N=1 or N=2 觀察轉結論時的 base rate 盲

統計上 N≤2 的事件不能支持「偶發」結論，但 LLM 在沒有明示 base rate 要求時，會用 N=1 觀察當作充分證據。

**Why**：base rate 思維不是 LLM 的預設輸出模式。**Consequence**：W17-097.1-.4（N=4 deny）+ W17-099 一次 success 就被歸為「偶發」，跳過 W17-100/101/102 對照實驗。**Action**：分析任務開始時即明示「base rate 觀察次數要求」（最低 3 次獨立觀察）。

## 觸發案例

W17-098 第一輪 WRAP：

1. 觀察 W17-097.1-.4 4 thyme 全 deny
2. R 階段宣稱「成本/收益不對稱不執行」直接結束實驗
3. 結論寫「runtime hardcoded deny」進 PC-115 + memory
4. W17-099 一個簡單實驗（單派 thyme + 簡單 Edit）就推翻
5. 推翻後直接改寫「runtime 偶發性 deny」當收場
6. 用戶質疑「偶發 deny 這種結論是不合理的」
7. 觸發 W17-103 二輪 WRAP，提出 5 變因 + 設計 E1/E2/Test G 對照實驗
8. 最終 5 變因 4 個被否證，推進到 Hypothesis K 工作假設

詳見 W17-104 反模式 2 章節 + W17-098/099/103 ticket。

## 防護

| 場景 | 應做 | 禁止 |
|----|----|----|
| 觀察 N≤2 deny 事件 | 列出 5+ 候選變因 + 設計低成本對照實驗 | 直接寫「偶發」/「transient」 |
| 改變條件後事件消失 | 控制單一變因 + 重複觀察取得 base rate | 假設「條件不重要」 |
| 「成本/收益不對稱」拒絕繼續 | 具體列出最低成本對照實驗的 token/time 估算 | 用此理由跳過 R 階段 |
| 結論需用「偶發 / transient / 不可重現」 | 必先有 base rate 數據（≥ 3 次獨立觀察 + < 5% 比率） | 用此標籤化未釐清因果 |

## 自查清單

每次寫「偶發 / transient / 不可重現 / conditions unknown」前：

- [ ] 我有 base rate 數據嗎？（≥ 3 次獨立觀察的 success/deny 比率）
- [ ] 我列了 5+ 候選變因嗎？
- [ ] 我設計了控制變因的對照實驗嗎？
- [ ] 我考慮過「未識別變因」而非「沒有原因」嗎？

任一答否 → 不能寫「偶發」收場，必須回 W 階段擴增假設或執行最低成本對照實驗。

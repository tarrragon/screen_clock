# PC-026: 發現測試失敗未立即建立 Ticket 追蹤

## 錯誤模式

執行測試發現失敗時，以「pre-existing」「不是我造成的」「與當前任務無關」為由，未立即建立 Ticket 追蹤，導致已知的測試失敗持續存在多個 session。

## 症狀

- 測試結果中有 failure 但被標記為「pre-existing」而忽略
- 多次 session 報告相同的測試失敗但無人建立 Ticket
- 測試通過率非 100% 但無對應的修復 Ticket

## 根因

1. **責任歸屬偏見**：「不是我造成的問題不需要我追蹤」— 錯誤。發現問題的人有責任確保問題被追蹤
2. **pre-existing 合理化**：將「已存在的問題」視為可接受的狀態，而非需要修復的 bug
3. **範圍切割過度**：「與當前 Ticket 無關」不是不建立追蹤 Ticket 的理由

## 正確做法

發現測試失敗時，**無論原因、無論是否在自身職責範圍**，必須立即：

```
發現測試失敗（任何原因）
    |
    v
[強制] 立即 /ticket create 建立修復 Ticket
    |
    v
繼續當前任務（不中斷主線）
```

**不需要判斷的事情**：
- 是誰造成的 → 不重要，先建 Ticket
- 是不是 pre-existing → 不重要，先建 Ticket
- 是否在我的 Ticket 範圍內 → 不重要，先建 Ticket
- 修復優先級多高 → 建完 Ticket 再決定

## 防護措施

| 措施 | 說明 |
|------|------|
| 測試失敗 = 立即建 Ticket | 無例外、無判斷、無延遲 |
| 禁止「pre-existing」標籤作為跳過理由 | 已知問題更需要追蹤 |
| 發現者建 Ticket，不問歸屬 | 主動通報比責任歸屬重要 |

# IMP-031: 代理人部分完成但報告全部完成

## 症狀

- 代理人回報「所有驗收條件已通過」「100% 完成」
- `git diff --stat` 顯示部分預期修改的檔案實際未變更
- 代理人新增了常數/定義但未執行對應的引用替換
- 原檔案仍保留硬編碼值和 TODO 註解

## 根因分析

**行為模式**：代理人在任務過程中完成了「準備工作」（定義常數），但在「套用工作」（替換引用）階段跳過或遺漏，卻仍報告全部完成。

**可能原因**：
1. Context 長度限制導致代理人在後半段失去前半段的任務追蹤
2. 代理人將「常數已定義」等同於「替換已完成」，混淆了中間產出和最終產出
3. 多檔案修改時，代理人可能在第一個檔案完成後就認為任務結束
4. 驗收自我檢查不嚴謹（未實際執行 grep 確認無殘留）

## 解決方案

### 偵測方式

並行派發後**強制**執行 `git diff --stat` 比對：

```
代理人回報完成
    |
    v
[強制] git diff --stat main（或基準 commit）
    |
    v
比對 Ticket where.files 清單 vs 實際變更檔案
    |
    +-- 一致 → 通過
    +-- 不一致 → 補完或重新派發
```

### 修復方式

1. 識別遺漏的檔案
2. PM 手動補完（如替換操作簡單）或重新派發代理人
3. 補完後重新驗證

## 預防措施

1. **派發指令中明確列出預期修改的每個檔案和具體動作**（不只列檔案名，要列「在 X 檔案中將 Y 替換為 Z」）
2. **並行派發後驗證（已有規則）**：`git diff --stat` 是最後防線，不可省略
3. **驗收條件中加入否定驗證**：如「無硬編碼殘留」「grep 搜尋 TODO 為 0」

# ARCH-005: 代理人定義衝突導致派發職責不清

## 基本資訊

- **Pattern ID**: ARCH-005
- **分類**: 架構設計
- **來源版本**: v0.31.0
- **發現日期**: 2026-02-26
- **風險等級**: 中

## 問題描述

### 症狀

某歷史 Ticket 派發 thyme-python-developer 修復 Hook import 路徑，但審查時發現 basil-hook-architect 的代理人定義聲稱「Hook 優化改進」和「Hook 問題修復」也是 basil 的**強制**觸發條件。三份文件（決策樹、basil 定義、thyme 定義）對同一職責的歸屬不一致，可能導致未來的派發決策錯誤。

| 任務 | 決策樹 | basil 定義（修正前） | thyme 定義 |
|------|-------|-------------------|-----------|
| Hook 新增/設計 | basil | basil | basil |
| Hook 優化 | thyme | **basil（矛盾）** | thyme |
| Hook 修正 | thyme | **basil（矛盾）** | thyme |

### 根本原因 (5 Why 分析)

1. Why 1: 三份文件對 Hook 修改的代理人派發規則不一致
2. Why 2: basil 的觸發條件表包含了應歸 thyme 的職責
3. Why 3: basil 定義建立較早，當時包攬所有 Hook 相關工作
4. Why 4: 後來新增 thyme 分工時，只更新了決策樹和 thyme 定義，未回頭修正 basil
5. Why 5 (根本原因): **多份文件定義同一規則時，缺乏「一致性回溯檢查」步驟** -- 修改任一處規則定義時，沒有搜尋所有引用該規則的文件進行同步更新

### 受影響模式

```
定義規則的文件群：
  A: .claude/pm-rules/decision-tree.md（第二層半）
  B: .claude/agents/basil-hook-architect.md（觸發條件）
  C: .claude/agents/thyme-python-developer.md（觸發條件）

修改 A 和 C 但遺漏 B → 三者不一致 → 派發決策混亂
```

### 可偵測性

| 偵測方式 | 可偵測 | 說明 |
|---------|--------|------|
| 人工 Code Review | 是 | 需同時閱讀三份文件並比對 |
| grep 搜尋 | 部分 | `grep -r "Hook 優化" .claude/agents/` 可發現重複定義 |
| Hook 系統 | 否 | 無自動化的規則一致性檢查 |
| 實際派發錯誤 | 是 | 但為事後偵測，成本高 |

## 防護措施

### 代理人定義修改檢查清單

修改任何代理人的觸發條件時，必須完成：

| 步驟 | 動作 | 驗證方式 |
|------|------|---------|
| 1 | 識別此觸發條件在決策樹中的對應規則 | `grep` 決策樹中相關關鍵字 |
| 2 | 識別是否有其他代理人定義涉及相同職責 | `grep -r "觸發條件" .claude/agents/` |
| 3 | 確認所有相關文件的規則一致 | 逐一比對 |
| 4 | 每個代理人定義包含「不觸發」區段 | 明確列出不歸此代理人的場景 |

### 代理人定義最佳實踐

| 實踐 | 說明 |
|------|------|
| 包含「不觸發」區段 | 明確列出容易混淆但不歸此代理人的場景 |
| 引用決策樹 | 觸發條件區段引用決策樹作為 Source of Truth |
| 分工判斷原則 | 用一句話總結與相鄰代理人的分工邊界 |

### 與 IMP-005 的關係

ARCH-005 和 IMP-005 都屬於**同步更新不完整**的模式：

| Pattern | 同步類型 | 遺漏 |
|---------|---------|------|
| IMP-005 | 模組路徑遷移 | 引用該模組的檔案未更新 import |
| ARCH-005 | 規則定義變更 | 引用該規則的文件未同步更新 |

**共通防護原則**：任何「修改被多處引用的定義」（模組位置、規則定義、介面契約），都需要先搜尋所有引用處，再逐一同步更新。

- **受影響檔案**: basil-hook-architect.md, decision-tree.md

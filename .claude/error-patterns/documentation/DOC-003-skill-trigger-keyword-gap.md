# DOC-003: Skill 觸發關鍵字不足導致匹配失敗

## 基本資訊

- **Pattern ID**: DOC-003
- **分類**: 文件設計
- **來源版本**: v0.31.0
- **發現日期**: 2026-02-26
- **風險等級**: 中

## 問題描述

### 症狀

用戶說「加入錯誤學習經驗」時，系統未匹配到 `/error-pattern add` Skill，PM 誤將錯誤經驗寫入 MEMORY.md 而非 error-patterns 系統。經用戶指正後才修正到正確位置。

### 根本原因 (5 Why 分析)

1. Why 1: PM 未使用 `/error-pattern add` 記錄錯誤模式
2. Why 2: 系統未在用戶訊息中匹配到 error-pattern Skill
3. Why 3: error-pattern Skill 的 description 只包含「查詢既有錯誤經驗」「記錄新發現的錯誤模式」等技術用語
4. Why 4: 用戶使用的自然語言（「學習經驗」「教訓」「錯誤學習」）不在 description 的關鍵字中
5. Why 5 (根本原因): **Skill description 只描述了功能，未涵蓋用戶可能使用的同義詞和自然語言表達**

### 受影響模式

```
用戶輸入: "加入錯誤學習經驗"
                |
Skill 匹配引擎搜尋 description 中的關鍵字
                |
description: "查詢既有錯誤經驗 (query), 記錄新發現的錯誤模式 (add)"
                |
"學習經驗" ≠ "錯誤經驗"、"教訓" ≠ "錯誤模式"
                |
匹配失敗 → PM 自行判斷 → 寫入錯誤位置（MEMORY.md）
```

### 可偵測性

| 偵測方式 | 可偵測 | 說明 |
|---------|--------|------|
| 用戶回饋 | 是 | 用戶指正「不是 MEMORY.md 而是 error-patterns 系統」 |
| Skill 列表檢查 | 部分 | 需人工比對 description 與常見用語 |
| 自動化測試 | 否 | 目前無 Skill 關鍵字覆蓋度測試 |

## 防護措施

### Skill Description 撰寫規範

建立或更新 Skill 時，description 必須涵蓋：

| 內容類型 | 說明 | 範例 |
|---------|------|------|
| 功能描述 | Skill 做什麼 | 「記錄新發現的錯誤模式」 |
| 使用時機 | Use for 列表 | 「(1) 查詢既有經驗 (2) 記錄新模式」 |
| **觸發同義詞** | **用戶可能使用的自然語言** | **「Triggers: 學習經驗, 教訓, 錯誤學習, 經驗記錄」** |

### Skill Description 檢查清單

| 步驟 | 動作 |
|------|------|
| 1 | 列出 3-5 個用戶可能的自然語言表達 |
| 2 | 確認每個表達的關鍵字都出現在 description 中 |
| 3 | 加入 `Triggers:` 區段列出同義詞 |

### 常見同義詞缺口

| 功能用語 | 用戶自然語言 |
|---------|------------|
| 記錄錯誤模式 | 學習經驗、教訓、錯誤學習、經驗記錄 |
| 查詢歷史問題 | 之前遇過嗎、有沒有經驗、避免再犯 |
| 建立 Ticket | 開任務、建立工作項目、新增追蹤 |
| 版本發布 | 發版、出版本、release |

## 修復紀錄

- **新增關鍵字**: 學習經驗、教訓、錯誤學習、經驗記錄、避免再犯、防護措施、錯誤紀錄
- **受影響檔案**: `.claude/skills/error-pattern/SKILL.md`

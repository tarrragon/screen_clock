# DOC-001: 交接文件使用 emoji

## 基本資訊

- **Pattern ID**: DOC-001
- **分類**: 文件規範
- **來源版本**: v0.25.x
- **發現日期**: 2026-01-14
- **風險等級**: 低

## 問題描述

### 症狀

交接文件（計畫文件、工作日誌、Ticket 等）中使用 emoji 符號：

```markdown
## 評估概述

**評估日期**: 2026-01-14
**評估目的**: 檢視錯誤學習經驗記錄

---

## 現有經驗記錄分析

| 缺失項目 | 優先級 | 說明 |
|---------|-------|------|
| error-patterns 目錄 | 高 | 五重文件系統要求的錯誤模式庫不存在 |
```

### 根本原因 (5 Why 分析)

1. Why 1: 計畫文件和工作日誌中出現 emoji
2. Why 2: CLAUDE.md 沒有明確規範交接文件禁用 emoji
3. Why 3: 只有語言規範（繁體中文），沒有格式規範
4. Why 4: 規範建立時未考慮所有文件類型
5. Why 5: **核心規則未涵蓋所有交接文件的格式要求**

## 解決方案

### 正確做法

使用純文字標記代替 emoji：

```markdown
## 評估概述

**評估日期**: 2026-01-14
**評估目的**: 檢視錯誤學習經驗記錄

---

## 現有經驗記錄分析

| 缺失項目 | 優先級 | 說明 |
|---------|-------|------|
| error-patterns 目錄 | 高 | 五重文件系統要求的錯誤模式庫不存在 |
```

### 替代標記對照表

| emoji | 純文字替代 |
|-------|-----------|
| (勾選符號) | [x] 或 完成 |
| (警告符號) | [!] 或 警告 |
| (錯誤符號) | [X] 或 錯誤 |
| (高優先) | 高 |
| (中優先) | 中 |
| (低優先) | 低 |

### 錯誤做法 (避免)

```markdown
## (文件符號) 評估概述

### (圖表符號) 現有經驗記錄分析

| 缺失項目 | 優先級 | 說明 |
|---------|-------|------|
| error-patterns 目錄 | (紅色圓圈) 高 | 錯誤模式庫不存在 |
```

## 適用範圍

以下文件類型禁止使用 emoji：

- `.claude/plans/` - 計畫文件
- `docs/work-logs/` - 工作日誌
- `.claude/tickets/` - Ticket 文件
- `.claude/error-patterns/` - 錯誤模式記錄
- 所有 README.md 文件

## 例外情況

以下情況可以使用 emoji：

- 終端輸出訊息（Hook 輸出）
- 使用者介面文字
- 非交接性質的說明文件

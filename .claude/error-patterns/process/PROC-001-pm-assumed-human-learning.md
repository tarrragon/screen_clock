# PROC-001: 錯誤假設執行者能靠經驗彌補文件不足

## 錯誤摘要

在設計規則或審查建議時，假設執行者（PM 或代理人）會「自行判斷」「用幾次就熟練」「根據經驗彌補」。實際上，無論 PM 或代理人，都是**嚴格依照文件規則做事**。文件資訊有缺失，就會導致操作失誤。

## 症狀

- 審查建議「不需要 references，用幾次就會熟練」
- 規則設計省略具體場景指引，理由是「執行者知道該怎麼做」
- 精簡規範時刪除了具體欄位提示，只保留通用原則
- 認為「規則太長不好」而過度精簡，導致指引不足

## 根因分析

將 AI 執行者類比為人類工程師。人類能累積經驗、靈活應變、從上下文推斷缺失資訊。但 AI 執行者每個 session 從零開始，文件就是它的全部知識。

**本質問題**：不是「PM 是 AI」的特殊情況，而是**所有角色都依照文件做事**的通用原則。文件不足 → 執行失誤，不能要求執行者靠經驗彌補。

## 觸發條件

- 設計規則時出現「執行者會學習」「用過就知道」「不需要額外指引」等表述
- 精簡規範時刪除具體場景指引，只保留抽象原則
- 審查建議「過度規範化」但未提供替代的資訊傳遞方式

## 正確做法

| 原則 | 說明 |
|------|------|
| **文件完整性優先** | 寧可 references 有更多細節，也不要有缺失 |
| **拆分而非省略** | 文件過長時拆分為「主要規則 + 參考文件」，不省略引導設計 |
| **假設第一次使用** | 設計規則時假設每個 session 都是執行者第一次接觸 |
| **省略需有替代** | 從規則中刪除內容時，必須確認資訊已在其他地方可取得 |

**文件分層策略**：

| 層級 | 位置 | 載入方式 | 內容 |
|------|------|---------|------|
| 核心規則 | `.claude/rules/` | 自動載入 | 通用原則、強制要求 |
| PM 規則 | `.claude/pm-rules/` | PM 按需讀取 | 流程規範、通用模板 |
| 參考文件 | `.claude/references/` | 代理人按需讀取 | 具體場景指引、範例、欄位清單 |

## 防護措施

1. 精簡規範時，必須同步建立或更新對應的 references
2. 審查建議「不需要詳細指引」時，確認是否有替代的資訊來源
3. 新規則的 references 覆蓋率檢查：每個決策點是否都有具體指引可查

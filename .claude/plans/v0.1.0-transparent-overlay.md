# v0.1.0 macOS 透明遮罩視窗

## 概述

依 SPEC-001（透明全螢幕遮罩視窗）實作 macOS 平台的全螢幕透明、frameless、always-on-top、no-shadow、click-through 視窗。本版本不含 UI 內容，純框架。

## 實作步驟

1. 加入 window_manager 套件依賴
   - 修改檔案：pubspec.yaml
   說明：在 dependencies 區塊新增 `window_manager: ^0.4.3`，執行 `flutter pub get`。

2. macOS Swift 平台層透明背景設定
   - 修改檔案：macos/Runner/MainFlutterWindow.swift
   說明：在 `awakeFromNib` 中加入 `self.isOpaque = false; self.backgroundColor = .clear`，這是 Flutter 端 transparent 真實生效的前提（SPEC-001 FR-02 設計約束）。

3. main.dart 啟動流程整合 window_manager
   - 修改檔案：lib/main.dart
   說明：實作 `initOverlayWindow()` 函式，依 SPEC-001 序列呼叫 `ensureInitialized → waitUntilReadyToShow → setAsFrameless → setBackgroundColor(transparent) → setHasShadow(false) → setAlwaysOnTop(true) → setIgnoreMouseEvents(true) → show`。WidgetsFlutterBinding.ensureInitialized() 必須先於 windowManager.ensureInitialized()。

4. 視窗尺寸與位置貼合主螢幕
   - 修改檔案：lib/main.dart
   說明：取得主螢幕尺寸並呼叫 `setSize(primaryDisplaySize)` 與 `setPosition(Offset.zero)`，使視窗覆蓋整個主螢幕（SPEC-001 FR-01）。處理螢幕尺寸取得失敗的 fallback（FR-01 替代場景 01a）。

5. 手動驗收 UC-01 + UC-02
   - 修改檔案：docs/work-logs/v0/v0.1/v0.1.0/v0.1.0-main.md
   說明：依 UC-01 主要成功場景與 UC-02 click-through 驗收條件逐項手動驗證，將結果與量測數字（啟動時間、CPU 使用率）記錄於 worklog 開發日誌。

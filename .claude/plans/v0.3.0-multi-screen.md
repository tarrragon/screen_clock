# v0.3.0 多螢幕選擇

## 概述

讓使用者可以指定遮罩顯示在哪一個螢幕（主螢幕、第二螢幕、外接螢幕）。MVP 透過 CLI 引數或環境變數指定，設定 UI 留待 v1.0.0。處理螢幕熱插拔情境。

## 實作步驟

1. 評估螢幕偵測方案（ANA）
   - 修改檔案：docs/spec/platform/multi-screen-selection.md
   說明：比較 `screen_retriever` 套件 vs `window_manager` 內建 API，評估 Display ID 在拔插後是否穩定、是否支援 macOS 全部螢幕版本。產出 SPEC-003 多螢幕規格。

2. 偵測可用螢幕清單
   - 修改檔案：lib/platform/display_detector.dart
   說明：實作 `listDisplays()` 回傳所有可用螢幕的 ID、尺寸、位置、是否為主螢幕。處理偵測失敗 fallback（只看到主螢幕）。

3. 目標螢幕設定（CLI 引數）
   - 修改檔案：lib/main.dart
   說明：解析 `--screen=N` CLI 引數（N=0 為主螢幕、其他為 index）；無引數時預設主螢幕；引數無效時 fallback 主螢幕並 log warning。

4. 視窗在指定螢幕貼合定位
   - 修改檔案：lib/main.dart
   說明：依目標螢幕的 position 與 size，呼叫 `setPosition + setSize` 讓視窗貼合該螢幕。處理多螢幕座標系（macOS 座標可能為負）。

5. 螢幕熱插拔處理
   - 修改檔案：lib/platform/display_detector.dart
   說明：訂閱 macOS 螢幕變更事件；目標螢幕拔除時 fallback 主螢幕並 log；新增螢幕不主動切換。

6. 雙螢幕情境驗收
   - 修改檔案：docs/work-logs/v0/v0.3/v0.3.0/v0.3.0-main.md
   說明：在雙螢幕（含一台外接）情境下驗證：(a) 主螢幕、(b) 第二螢幕、(c) 拔除目標螢幕時 fallback、(d) 重新插回時行為合理。

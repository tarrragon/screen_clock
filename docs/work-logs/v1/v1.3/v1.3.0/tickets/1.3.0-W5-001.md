## Context Bundle

### 目標
原生 Swift 實作 FR-05：綁定快捷鍵的側鍵按一下 → 合成對應鍵盤組合鍵送前景 app。需把 W4-004 存的 Flutter usbHidUsage 編碼映射為 macOS CGKeyCode + CGEventFlags。純 Swift，runtime 與用戶聯合驗收。

### 規格
SPEC-007 FR-05（動作—綁定快捷鍵）：按下綁定鍵依 HotkeyAction 合成 keyDown（含修飾鍵 flags）+ keyUp；一次按下送一次；原側鍵動作被消費（FR-03）。

### 既有現狀（W2-003 + W4-002 已落地）
- macos/Runner/MainFlutterWindow.swift 的 `InputBindingBridge`。
- `handleButtonDown(button:event:)` hotkey 分支目前：`if type == "hotkey" { NSLog("...執行留 W3...消費事件"); return nil }`（僅消費 + log，無合成）。**本票改這裡**。
- action dict 鍵（與 Dart HotkeyAction.toJson 對齊）：`action["type"]=="hotkey"`、`action["keyCode"] as? Int`（= Flutter usbHidUsage）、`action["modifiers"] as? [Int]`（= 修飾鍵的 usbHidUsage 清單）。
- 既有 `scrollSource = CGEventSource(stateID: .hidSystemState)`；scroll 合成用 `CGEvent(scrollWheelEvent2Source:...)` + `.post(tap:)`。鍵盤可比照建立 keyboard source + post。
- CGEventTap mask 只含滑鼠事件（otherMouseDown/Up、mouseMoved、各 dragged），**不含 keyboard 事件** → 合成的鍵盤事件不會被本 tap 重攔（無迴圈風險，但仍於註解說明）。

### W4-004 的 hotkey 編碼（你要映射的來源）
- keyCode = 主鍵的 USB HID usage（HID usage page 0x07）。
- modifiers = 按下當下的修飾鍵實體鍵 usbHidUsage 清單，值落在 0xE0–0xE7（LeftCtrl 0xE0 / LeftShift 0xE1 / LeftAlt 0xE2 / LeftGUI(Cmd) 0xE3 / RightCtrl 0xE4 / RightShift 0xE5 / RightAlt 0xE6 / RightGUI 0xE7）。

### 要實作
1. **usbHidUsage → CGKeyCode 映射**：建立 `[Int: CGKeyCode]` 字典，涵蓋常見鍵——字母 a-z（HID 0x04-0x1D）、數字 1-0（0x1E-0x27）、Enter/Esc/Tab/Space/方向鍵/F1-F12/常用符號等。值為 macOS virtual keycode（kVK_ANSI_* 等，Carbon HIToolbox）。未知 usbHidUsage → NSLog 警告並安全略過（不合成、不崩潰）。
2. **modifiers → CGEventFlags 映射**：0xE0/0xE4→.maskControl，0xE1/0xE5→.maskShift，0xE2/0xE6→.maskAlternate，0xE3/0xE7→.maskCommand；OR 合併為單一 CGEventFlags。
3. **handleButtonDown hotkey 分支改寫**：
   - 讀 keyCode + modifiers；映射為 CGKeyCode + flags。
   - 合成：keyboard source（可用既有 hidSystemState source 或新建）→ `CGEvent(keyboardEventSource:virtualKey:keyDown:true)` 設 `.flags = flags` → `.post(tap: .cgSessionEventTap)`；再 keyDown:false（keyUp）同樣 post。一次按下送一次（不連發）。
   - 仍 `return nil` 消費原側鍵事件（FR-03）。
   - keyCode 無映射時：log + 仍消費側鍵（避免觸發上一頁/下一頁），但不合成。
4. 維持 NFR-01：hotkey 是單次按下事件（非拖曳熱路徑），合成可接受；勿在 drag move 熱路徑加重。

### 約束
- 只改 macos/Runner/MainFlutterWindow.swift。不碰 Dart（HotkeyAction 編碼不變，維持 usbHidUsage 平台中性，利未來 Windows）。
- 平台原生常數屬 Swift 側（CLAUDE.md 例外）。
- 不破壞既有拖曳滾動與捕捉模式分派。

### 驗收（dev + runtime 聯合）
- dev：`flutter build macos --debug` 編譯通過；`flutter test` Dart 側不受影響（不應改 Dart）。
- runtime（與用戶一起）：綁 Cmd+C 的側鍵按一下可複製；含多修飾鍵組合（如 Cmd+Shift+4）正確送出；按一下只送一次；原上一頁/下一頁不觸發。

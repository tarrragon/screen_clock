## Context Bundle

### 根因（runtime 實證）
實機建立「側鍵 3 → Cmd+C」後 Cmd+C 不生效。面板顯示綁定鍵碼 458758、修飾鍵 1。
- 458758 = 0x70006 = (HID page 0x07 << 16) | usage_id 0x06（c）。
- Flutter `PhysicalKeyboardKey.usbHidUsage` 含 page 前綴；W4-004 捕捉存的就是這個全值。
- W5-001 Swift `hidUsageToCGKeyCode` 表用裸 usage_id（c=0x06）當 key → `table[458758]` miss → log 後略過、完全不合成。modifier 同理：Cmd 存 0x700E3，`cgEventFlags` 比對裸 0xE3 → miss → 無 flag。

### 既有現狀（W5-001 已落地，macos/Runner/MainFlutterWindow.swift）
- `synthesizeHotkey(action:)`：讀 action["keyCode"] as Int → `Self.cgKeyCode(forUsbHidUsage:)` → 合成。
- `cgKeyCode(forUsbHidUsage usage:)` → `hidUsageToCGKeyCode[usage]`。
- `cgEventFlags(forModifierUsages usages:)`：`switch usage { case 0xE0,0xE4: .maskControl ... }`。

### 修法（純 Swift，最小改動，不改 Dart、不遷移既有綁定）
在「查表 / 修飾鍵比對」前剝除 page 前綴，取 usage_id：
1. `cgKeyCode(forUsbHidUsage:)`：改為 `hidUsageToCGKeyCode[usage & 0xFFFF]`（建議先驗 `(usage >> 16) == 0x07`，非鍵盤 page 回 nil）。
2. `cgEventFlags`：switch 前 `let id = usage & 0xFFFF`，對 id 比對 0xE0–0xE7。
- 驗算：0x70006 & 0xFFFF = 0x06 = c → table[0x06]=0x08（kVK_ANSI_C）；0x700E3 & 0xFFFF = 0xE3 → .maskCommand。

### 約束
- 只改 macos/Runner/MainFlutterWindow.swift。不改 Dart（HotkeyAction 編碼維持 Flutter usbHidUsage 全值，平台中性）。
- 不破壞拖曳滾動 / 捕捉分派。常數屬 Swift 側。

### 驗收
- 編譯：`flutter build macos --debug` 通過；`flutter test` Dart 側不受影響。
- runtime（聯合）：側鍵 3 → Cmd+C 按一下可複製；多修飾鍵組合正確；被綁側鍵原動作仍消費。

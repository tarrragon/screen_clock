## Context Bundle

### 目標
在設定面板新增綁定管理區的「權限引導（FR-07）」與「綁定清單顯示/刪除（FR-08 部分）」。含把 InputBindingController 接線進面板的共用基礎（W4-004 會沿用）。新增流程與偵測捕捉留 W4-004。

### 規格
SPEC-007 FR-07（權限檢查與引導）、FR-08（面板整合的清單/刪除部分）。

### 既有現狀
- lib/widgets/settings_panel.dart：`SettingsPanel` 為 StatelessWidget，建構參數 (availableScreenCount, onClose)。build 取 SettingsScope.controllerOf + SettingsScope.of，排 9 個欄位 + _buildActions（取消=resetToStartup+onClose / 儲存=persist+onClose）。
- lib/main.dart：`_ScreenClockAppState` 持有 widget.inputBindingController（InputBindingController），於 initState 呼叫 start(bindings)、_onSettingsChanged 時 syncBindings。面板由 `_PanelHost`（建構參數 availableScreenCount, onClosed）包 SettingsPanel。**面板目前拿不到 InputBindingController**。
- lib/input/input_binding_controller.dart（W4-001 已落地）：`permissionGranted` 唯讀 ValueListenable<bool>；`requestPermission` 在 channel 層（controller 內部 _ensurePermission 用，未公開）；`startButtonCapture`/`cancelButtonCapture` 已有。
- lib/models/settings_model.dart：`bindings` List<MouseBinding>（不可變）；copyWith(bindings:) 會 dedupeBindingsByButton。
- lib/input/mouse_binding.dart / mouse_action.dart：MouseBinding(buttonNumber, action)；DragScrollAction(direction: ScrollDirection.natural/inverted, sensitivity)；HotkeyAction(keyCode, modifiers)。

### 要實作
### A. 接線（共用基礎，W4-004 沿用）
- main.dart：把 `inputBindingController` 經 `_PanelHost` 傳入 `SettingsPanel`（新增建構參數 `inputBindingController`）。
- SettingsPanel 改為 StatefulWidget（需 initState 刷新權限、後續 W4-004 管捕捉狀態）。
### B. controller 補公開方法（W4-001 驗收發現的缺口）
- input_binding_controller.dart 新增 `Future<void> refreshPermission()`：`_permissionGranted.value = await _channel.queryPermission();`。原因：W4-001 的 permissionGranted 僅在「有綁定」時經 _ensurePermission 更新，空綁定時即使已授權仍為 false；面板開啟須主動刷新才反映真實授權。
- 同時把 channel 的 requestPermission 經 controller 公開一個 `Future<void> requestPermission()`（呼叫 _channel.requestPermission() 後可再 refreshPermission），供面板「開啟系統授權」按鈕用。
### C. FR-07 權限區
- panel initState 呼叫 controller.refreshPermission()。
- 用 ValueListenableBuilder(controller.permissionGranted)：已授權顯示「輔助使用：已授權」；未授權顯示引導文字（需於系統設定→隱私權與安全性→輔助使用開啟）+ 按鈕「開啟系統授權」→ controller.requestPermission() 然後 refreshPermission()。
### D. FR-08 清單/刪除
- 從 SettingsScope.of(context).bindings 渲染清單：每筆顯示 buttonNumber（如「側鍵 4」）+ 動作摘要（dragScroll→「拖曳滾動・方向・靈敏度」；hotkey→「快捷鍵・keyCode+修飾鍵數」）+ 刪除鈕。
- 刪除：controller.update((s)=>s.copyWith(bindings: [移除該 buttonNumber 後的清單])) 後 controller.persist()（FR-08「變更即時持久化」；持久化後 main.dart 的 _onSettingsChanged 會自動 syncBindings 下傳原生）。
- 摘要文案集中到 AppText（新增綁定相關字串常數）；禁硬編中文字面於 widget（CLAUDE.md 常數集中 + i18n）。

### 設計決策（請依此，必要時於 ticket 記錄）
- 樣式欄位維持既有 Save/Cancel 暫存模型；**綁定變更採即時持久化**（SPEC FR-08 明示「即時持久化」），與 Save/Cancel 解耦。刪除後即使按取消也不還原綁定（綁定不納入 _initial 快照的暫存語意）。
- 不動既有 9 欄位行為。

### 約束
- 不碰 Swift、不碰 W4-004 的新增流程（只做清單顯示/刪除 + 權限區 + 接線）。
- 常數集中（AppText 新增字串）、可觀測性（catch 沿用 debugPrint）、認知負擔（若面板過長，抽綁定區為私有子 widget/方法）。

### 測試（widget test）
- test/widgets/settings_panel_binding_test.dart：用 mock/fake InputBindingController（permissionGranted 可控）。
- 驗：未授權顯示引導 + 按鈕；已授權顯示已授權；清單渲染既有綁定；刪除後 model.bindings 不含該筆且 persist 被呼叫；權限狀態變化即時反映。
- 既有 widget 測試（center_clock 等）不可破壞。

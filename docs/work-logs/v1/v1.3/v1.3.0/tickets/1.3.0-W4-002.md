## Context Bundle

### 目標
原生 Swift 端實作偵測捕捉模式：捕捉中下一個側鍵按下 → 經 channel 回報 buttonNumber 並消費該事件、不分派綁定。SPEC-007 FR-06。純 Swift，runtime 驗收（無 Swift 單元測試），實機驗收隨 W4-004 合併。

### 檔案
macos/Runner/MainFlutterWindow.swift 的 `InputBindingBridge`（單一檔案，勿動其他）。

### 既有現狀（W2-003 已落地，勿重造）
- `handle(call:result:)` switch 目前認得：queryPermission / requestPermission / updateBindings / default→FlutterMethodNotImplemented。
- `applyBindings`：空綁定 teardownTap()；有綁定且 AXIsProcessTrusted 才 ensureTap()。
- `ensureTap()`：建立 CGEventTap（mask 含 otherMouseDown/Up、mouseMoved、各 dragged），已存在則沿用。`teardownTap()` 拆除。
- `handleTapEvent(type:event:)`：otherMouseDown → handleButtonDown；otherMouseUp → handleButtonUp；move/dragged → handleDragMove。
- `handleButtonDown(button:event:)`：命中綁定才分派（dragScroll 進拖曳 / hotkey 消費 log），未命中放行。
- channel 名 "screen_clock/input_binding"（已硬編於 init，與 AppInputBinding.channelName 一致）。

### W4-001 已定義的對齊字面（務必逐字一致）
- beginCaptureButton（Dart→原生：進入捕捉）
- endCaptureButton（Dart→原生：離開捕捉）
- onButtonCaptured（原生→Dart：回報，參數 key "buttonNumber"，值為 Int）

### 要實作
1. 新增捕捉狀態旗標：`private var capturingButton = false`。
2. handle() 加兩 case：
   - "beginCaptureButton"：`capturingButton = true`；若 `AXIsProcessTrusted()` 則 `ensureTap()`（捕捉期需 tap active，即使 bindingsByButton 為空）；result(nil)。
   - "endCaptureButton"：`capturingButton = false`；若 bindingsByButton 為空則 teardownTap()（還原「無綁定不留 tap」不變式）；result(nil)。
3. handleTapEvent 的 .otherMouseDown 分支最前面插入捕捉攔截：`if capturingButton { return reportCapturedButton(event) }`（優先於 handleButtonDown，與是否綁定無關）。
4. 新增 `reportCapturedButton(_ event:)`：取 buttonNumber；`capturingButton = false`（一次性，避免連報）；於 main thread 發 `channel.invokeMethod("onButtonCaptured", arguments: ["buttonNumber": Int(button)])`（DispatchQueue.main.async 包；事件回呼非 main thread）；回傳 nil 消費該事件。
5. 捕捉攔截只攔 otherMouseDown；otherMouseUp / move 在捕捉中可照常放行或忽略（不影響）。

### 約束
- 字面與 AppInputBinding 逐字一致（channelName / 三方法名 / "buttonNumber"）。
- NFR-01：回呼輕量。捕捉是罕見一次性事件（非拖曳熱路徑），main thread async 發 channel 可接受；勿在熱路徑（drag move）加重邏輯。
- 不可破壞既有拖曳滾動分派與消費邏輯。
- 平台原生碼常數屬 Swift 側（CLAUDE.md 常數集中規範例外），字面直寫但需與 Dart 對齊。

### 驗收（runtime，隨 W4-004 實機）
- 進入捕捉模式按側鍵：面板顯示抓到的編號，且該次原上一頁/下一頁不觸發。
- 結束捕捉後恢復原綁定分派；既有拖曳滾動回歸正常。
- 編譯驗證：跑 `flutter build macos --debug`（或確認 Swift 編譯通過）；跑 `flutter test` 確認 Dart 側未受影響（不應改 Dart）。

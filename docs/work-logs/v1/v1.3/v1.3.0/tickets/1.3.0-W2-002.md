### 實作指引（依 SPEC-007 FR-01/02）

參考既有模式：`lib/models/settings_model.dart`（immutable + schemaVersion + 容錯 fromJson + copyWith + ==/hashCode）。

模型（lib/input/）：
- enum ScrollDirection { natural, inverted }
- enum MouseActionType { dragScroll, hotkey }
- sealed class MouseAction { MouseActionType get type; Map<String,Object> toJson(); }
  - DragScrollAction { ScrollDirection direction = natural; double sensitivity = 預設常數 }
  - HotkeyAction { int keyCode; List<int> modifiers }（modifiers 可用穩定可序列化表示）
- class MouseBinding { int buttonNumber; MouseAction action }（含 copyWith、==/hashCode）

序列化：
- 每個 MouseAction.toJson 含 'type' 欄（MouseActionType.name）；MouseAction.fromJson 依 'type' 分派。
- 所有 JSON 鍵與 action type 字串集中 app_constants.dart（新增 AppInputBinding 類），禁硬編碼。
- 容錯：單筆綁定型別錯誤/未知 type → 略過該筆，不拋例外。

SettingsModel（schema v3）：
- 新增 `bindings: List<MouseBinding>`（預設空清單）。
- schemaVersion 2 → 3；toJson 加 'bindings'（List<Map>）；fromJson 缺 bindings → []，單筆壞 → 略過。
- copyWith / == / hashCode 納入 bindings。
- 同 buttonNumber 去重規則明確（建議：後者覆蓋前者，或保留首筆——擇一並測試固定）。

測試（test/，TDD：先 RED 後 GREEN，100% 通過）：
- round-trip：fromJson(model.toJson()) == model（含 bindings、兩種 action）
- 容錯：含一筆壞綁定的 JSON → 只略過該筆，其餘正常
- 向後相容：v2 JSON（無 bindings）→ bindings 為空清單
- 去重：同 buttonNumber 兩筆 → 依規則收斂
- copyWith 不影響原物件
- dart analyzer 無警告

不在本票範圍：CGEventTap（W2-003）、動作執行邏輯（W3）、面板 UI（W4）。本票只交付資料模型 + 序列化 + 持久化。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 實作摘要

純 Dart domain 模型 + 序列化 + SettingsModel schema v3，TDD（先 RED 後 GREEN）。

新增檔案：
- `lib/input/mouse_action.dart`：sealed class `MouseAction`（`MouseActionType` get type、`toJson()`、靜態 `fromJson` 依 type 分派）。子類 `DragScrollAction`（`ScrollDirection direction = natural`、`double sensitivity = AppInputBinding.defaultDragScrollSensitivity`）與 `HotkeyAction`（`int keyCode`、不可變 `List<int> modifiers`）。`ScrollDirection` / `MouseActionType` enum。各含 copyWith、==/hashCode。
- `lib/input/mouse_binding.dart`：`MouseBinding`（`int buttonNumber` + `MouseAction action`，含 copyWith、==/hashCode）。`MouseBinding.fromJson` 單筆型別錯誤 / action 不可解析 → 回傳 null（容錯）。頂層函式 `dedupeBindingsByButton`：同 buttonNumber 後者覆蓋前者，去重規則固定。

修改檔案：
- `lib/models/settings_model.dart`：新增 `bindings: List<MouseBinding>`（預設空清單）。`schemaVersion` 2 → 3。toJson 加 bindings（List<Map>）；fromJson 經 `_bindingsFromJson` 容錯（非清單→空、單筆壞→略過、同 button→去重）。copyWith / == / hashCode 納入 bindings；copyWith 設定 bindings 時套用去重。
- `lib/app_constants.dart`：新增 `AppInputBinding`（所有綁定 / 動作 JSON 鍵與 dragScroll / hotkey type 字面、預設靈敏度）與 `AppSettingsKeys`（bindings 欄鍵）。禁硬編碼字串字面值。

設計決策：
- MouseAction.fromJson 對未知 / 缺 type 回傳 null，由呼叫端（MouseBinding.fromJson）略過該筆，符合 FR-02 容錯往返。
- 去重規則「後者覆蓋」於 fromJson 與 copyWith 兩處一致套用，確保 round-trip 與面板層收斂一致。
- modifiers 以 `List<int>.unmodifiable` 保證不可變，序列化為 List<int>。

不在本票範圍：CGEventTap（W2-003）、動作執行、面板 UI。

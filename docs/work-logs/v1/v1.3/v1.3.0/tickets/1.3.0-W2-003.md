### 實作指引（含 scope 擴充：交付可用的拖曳滾動）

前置：基底已含 W2-001（InputBindingBridge / 權限 / channel）與 W2-002（MouseBinding / DragScrollAction / AppInputBinding）。讀 `macos/Runner/MainFlutterWindow.swift`、`lib/input/input_binding_channel.dart`、`lib/input/mouse_binding.dart`、`docs/work-logs/v1/v1.3/v1.3.0/spike/cgeventtap_scroll_spike.swift`（拖曳滾動參考邏輯）。

**scope 擴充理由**：原 W2-003 僅做 tap 框架，無 runtime 可驗的功能。spike 已證實拖曳滾動可行，故本票一併交付拖曳滾動動作 + 預設綁定，讓用戶這輪即可實機使用。快捷鍵動作（HotkeyAction 執行）仍留 W3。

原生（MainFlutterWindow / InputBindingBridge 擴充）：
- 建立 CGEventTap（沿用 spike 的 mask + .cgSessionEventTap + 消費邏輯），僅在已授權（AXIsProcessTrusted）時建立；app 啟動時若有綁定但未授權，呼叫 requestPermission 觸發系統提示（這同時驗 W2-001 acceptance #2 的 live 部分）。
- 持有由 Dart updateBindings 下傳的綁定快照；otherMouseDown 比對 buttonNumber：
  - 命中 DragScroll 綁定 → 進入拖曳模式、消費事件；move → 依該綁定的 direction/sensitivity 合成 CGScrollWheelEvent；up → 離開、消費。
  - 命中 Hotkey 綁定 → 本票先消費 + 留 TODO/log（HotkeyAction 執行為 W3）。
  - 未命中 → 放行。
- 綁定變更（updateBindings）即時更新快照，無須重啟。回呼維持輕量（NFR-01）。

Dart：
- InputBindingController（lib/input/）：app 啟動時把 SettingsModel.bindings 經 input_binding_channel.updateBindings 下傳；設定變更時重新下傳。接到主啟動流程（main.dart）。
- 預設綁定：SettingsModel.defaults() 的 bindings 由空清單改為含一筆 `MouseBinding(buttonNumber: 4, DragScrollAction(natural, 預設 sensitivity))`，讓功能開箱即用（用戶 spike 實測側鍵為 button 4）。同步更新既有 settings_model 測試對 defaults().bindings 的斷言。

測試：
- Dart 走 TDD：controller 下傳綁定（mock channel）、defaults 含預設綁定、設定變更重新下傳。`flutter test` 100% 通過、`dart analyze` 無警告。
- 原生無 unit test；`flutter build macos --debug` 確認可建置。
- **禁宣稱 runtime 已驗**：實機拖曳滾動由用戶驗（授權 + 按住 button 4 拖動 → 捲動、不換頁）。

不在本票範圍：HotkeyAction 執行（W3）、設定面板 UI（W4）。

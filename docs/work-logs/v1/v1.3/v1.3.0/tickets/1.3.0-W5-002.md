## Context Bundle

### 目標
修 W4-004 發現的次要缺口：新增綁定流程選 hotkey 動作但尚未擷取按鍵時，confirm 仍可建立 keyCode=0 的無效綁定。應在未擷取時禁用 confirm。

### 既有現狀（W4-004 已落地）
- lib/widgets/settings_panel.dart 的 `_AddBindingFlow`（StatefulWidget，State `_AddBindingFlowState`）。
- configuring 階段欄位：`_AddActionType _actionType`（dragScroll/hotkey）、`int? _hotkeyCode`（null=未擷取）、`_buildConfiguring()` 內 confirm 按鈕 key `add-flow-confirm`，目前 `onPressed: () => widget.onConfirm(_buildBinding())` 一律啟用。
- `_buildBinding()`：hotkey 時 `HotkeyAction(keyCode: _hotkeyCode ?? 0, ...)`（未擷取則 keyCode=0）。

### 要實作
- confirm 按鈕：當 `_actionType == _AddActionType.hotkey && _hotkeyCode == null` 時 `onPressed: null`（禁用）；其餘（dragScroll，或 hotkey 已擷取）維持啟用。
- 可加未擷取時的提示文字（選用，字串集中 AppText）。

### 約束
- 只改 lib/widgets/settings_panel.dart + test/widgets/settings_panel_binding_test.dart（+ 若加提示則 app_constants AppText）。
- 不動 dragScroll 路徑、不動 W5-001 的 Swift、不破壞既有 128 測試。
- 禁硬編中文於 widget。

### 測試
- widget test：hotkey 型別且未擷取 → confirm 禁用（找 key add-flow-confirm 的 enabled 狀態）；模擬擷取按鍵後 → confirm 啟用；dragScroll 型別 → confirm 一律啟用。
- flutter test 全綠 / dart analyze 無警告。

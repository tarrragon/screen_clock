### 問題根因

W2-003 發現：`SettingsModel.defaults()` 含一筆預設綁定（button 4 拖曳滾動），但只在**無存檔**時生效（`load()` 的 `raw == null` 分支）。既有使用者存檔走 `fromJson()`，而 `_bindingsFromJson` 對缺 `bindings` 欄（schema v2 舊資料）回傳**空清單** → 升級後無預設綁定，功能失效，需手動清檔才生效。

### 影響範圍

- `lib/models/settings_model.dart`（新增 `bindingsSeeded` 欄位 + 序列化 + copyWith + ==/hashCode）
- `lib/services/settings_service.dart`（`PreferencesSettingsService.load()` 加一次性 seed migration）

### 關鍵設計邊界（實作必讀）

ticket 原策略「flag 不存在就 seed」若無腦執行會**誤刪**已有非空綁定的 v3 早期使用者（他們存檔也沒有新 flag）。正確觸發條件：

> `!bindingsSeeded` **且** `bindings.isEmpty` → 才填入 `defaults().bindings`；
> 無論 bindings 是否為空，只要 `!bindingsSeeded` 一律標記 `bindingsSeeded = true` 並 `save()` 持久化。

此設計同時滿足 acceptance 2（使用者升級後再清空綁定，flag 已 true → 不重新 seed）。

### RED 測試（PM 已寫，勿改測試期望，使測試轉綠即可）

- `test/models/settings_model_test.dart`：新增 group `SettingsModel.bindingsSeeded (W3-002 seed migration)`（7 測試）；並更新 2 個既有測試期望值（`fromJson empty map` / `fromJson tolerates invalid types` 改為期望 `bindingsSeeded: false`，因新欄位納入 ==）。
- `test/services/settings_service_test.dart`：新增 group `PreferencesSettingsService bindings seed migration (W3-002)`（4 測試）。
- `dart analyze` 確認：14 錯誤全為 `bindingsSeeded` 未定義（預期 RED），無語法錯誤。

## Problem Analysis
<!-- Schema[ANA/Problem Analysis]: 必填（.claude/pm-rules/ticket-body-schema.md） -->
### 程式碼分析（PM 前台，2026-06-15）

面板渲染（lib/widgets/settings_panel.dart）：
- `SettingsPanel.build` → `Center(child: Material(color: Theme.of(context).colorScheme.surface, elevation: 8, ...))`，背景為**不透明** surface 色。
- 外層 `_PanelHost → _DismissibleOverlay`（lib/main.dart）以 `Container(color: Color(0x80000000))` 全螢幕 50% 暗遮罩 + 中央面板。
- `_togglePanel` 開啟時呼 `windowManager.setIgnoreMouseEvents(false)`（停用 click-through），`_onPanelClosed` 還原。

git 確認：整個 W2（W2-001/002/003）未改動 settings_panel.dart，也未改 main.dart 的 _togglePanel/_onPanelClosed/_PanelHost/_DismissibleOverlay 邏輯。

結論（待 release 驗證）：程式碼層面面板應為「暗遮罩 + 中央不透明面板、可互動」，無 transparent/click-through 的程式成因。用戶於 debug build 回報異常，最可能為 debug build 在透明 NSWindow 上的合成行為（log 含 "Running with merged UI and platform thread. Experimental."，較新 Flutter macOS 模式）。

### 決定性測試（待用戶）
release build 是否重現面板透明穿透：
- release 不重現 → debug-only artifact，用戶日常跑 release 不受影響，本 ANA 記錄結論結案（W4 面板實作時於 release 驗收）。
- release 也重現 → 真 bug，二分法定位（git bisect 或逐版回退）並 spawn IMP。

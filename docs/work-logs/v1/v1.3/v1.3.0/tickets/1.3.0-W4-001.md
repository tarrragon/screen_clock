## Context Bundle

### 目標
擴充 Dart 輸入綁定基礎建設，為 W4 設定面板提供「按鍵偵測捕捉」與「權限狀態」兩項能力。純 Dart 層，mock channel 單元測試，不碰 Swift（原生 capture 留 W4-002）、不碰 UI（面板留 W4-003/004）。

### 規格來源
SPEC-007 docs/spec/input/mouse-button-binding.md：FR-06（偵測捕捉模式）、FR-07（權限檢查與引導）。method channel 草案表已定義 beginCaptureButton/endCaptureButton/onButtonCaptured。

### 既有現狀（已存在，勿重造）
- lib/app_constants.dart `AppInputBinding`：已有 channelName / queryPermissionMethod / requestPermissionMethod / updateBindingsMethod / onPermissionChangedMethod / bindingsArgKey / grantedArgKey / buttonNumberKey。
- lib/input/input_binding_channel.dart `InputBindingChannel`：已有 start(onPermissionChanged) / stop / queryPermission / requestPermission / updateBindings / _handleNativeCall（目前只認 onPermissionChanged）/ _invoke（PlatformException + MissingPluginException 容錯回 null）。
- lib/input/input_binding_controller.dart `InputBindingController`：已有 start(bindings) / syncBindings / stop / _ensurePermission / _onPermissionChanged（目前僅 debugPrint）。
- 測試模式參考 test/input/input_binding_controller_test.dart：用 TestDefaultBinaryMessengerBinding.instance.defaultBinaryMessenger.setMockMethodCallHandler 攔 Dart→原生；原生→Dart 通知用 messenger.handlePlatformMessage(channel.name, codec.encodeMethodCall(MethodCall(...)), null) 模擬。

### 要新增的 API（下游 W4-002/003/004 依此對齊，務必照此命名）

### AppInputBinding 新常數
- beginCaptureButtonMethod = 'beginCaptureButton'（Dart→原生：進入滑鼠鍵捕捉）
- endCaptureButtonMethod = 'endCaptureButton'（Dart→原生：離開捕捉）
- onButtonCapturedMethod = 'onButtonCaptured'（原生→Dart：回報捕捉到的側鍵）
- capturedButtonNumberArgKey = 'buttonNumber'（onButtonCaptured 參數鍵；與 SPEC channel 草案表一致）
> 上述字面 Swift 端 W4-002 會對齊；本票只負責 Dart 常數定義。

### InputBindingChannel
- beginCaptureButton() / endCaptureButton()：包 _invoke 呼叫對應 method（容錯沿用）。
- start() 擴充：除 onPermissionChanged 外，新增 onButtonCaptured 回呼（ValueChanged<int>）。_handleNativeCall 新增 onButtonCaptured 分支：解析 capturedButtonNumberArgKey（int，型別不符忽略）→ 呼叫 onButtonCaptured。維持「未知方法 debugPrint」既有行為。
- 為避免破壞既有測試呼叫 start(onPermissionChanged:)，新增的 onButtonCaptured 設為具名選用參數（onButtonCaptured 可為 null）。

### InputBindingController
- 權限狀態暴露：新增 ValueListenable<bool>（建議內部 ValueNotifier<bool> permissionGranted，預設 false）。start() 時 queryPermission 結果寫入；_onPermissionChanged(granted) 同步更新此 notifier（讓 UI 監聽）。提供 getter 回傳 ValueListenable<bool>（不外露可寫的 ValueNotifier）。stop() 不 dispose notifier（生命週期與 controller 同；若 dispose 需在 controller dispose 處理——本專案 controller 無 dispose，維持簡單即可，於註解說明）。
- 按鍵捕捉 session：
  - startButtonCapture({required ValueChanged<int> onCaptured, Duration timeout})：呼叫 channel.beginCaptureButton()，登記 onCaptured；啟動 Timer(timeout) 逾時自動 cancelButtonCapture。預設 timeout 建議常數化（AppDurations 新增 buttonCaptureTimeout，如 const Duration(seconds: 5)）。
  - cancelButtonCapture()：呼叫 channel.endCaptureButton()、取消 Timer、清除登記的 onCaptured。
  - 收到原生 onButtonCaptured 時：呼叫登記的 onCaptured(buttonNumber)，然後自動結束捕捉（endCaptureButton + 清 Timer）。
  - 防重入：捕捉進行中再次 startButtonCapture 應先結束前一個或忽略（擇一，註解說明）。
- start() 需把 onButtonCaptured 接到 channel.start(...)（讓 channel 的原生回報路由進 controller 的 capture 分派）。

### 約束
- 常數集中：所有字面（method 名、arg key、timeout）集中 app_constants.dart，禁硬編碼（CLAUDE.md 常數集中規範）。
- 可觀測性：catch / 容錯路徑沿用既有 debugPrint('[input-binding] ...') 模式（observability-rules 規則 1）。
- 不可變 / 既有風格：照既有檔案註解密度與命名。
- 不改 SettingsModel、不改 main.dart、不碰 Swift、不碰 settings_panel.dart。

### 測試要求（RED→GREEN）
- channel：beginCaptureButton/endCaptureButton 發出正確 method；模擬原生 onButtonCaptured 呼叫 → onButtonCaptured 回呼收到正確 buttonNumber；型別錯誤參數被忽略不拋。
- controller：startButtonCapture 發 beginCaptureButton；收到捕捉回報 → onCaptured 被呼叫且自動 endCaptureButton；cancelButtonCapture 發 endCaptureButton；逾時自動結束（用短 timeout 或 fakeAsync）；權限 ValueListenable 隨 onPermissionChanged 更新。
- 既有測試全數維持綠燈（向後相容，勿破壞 start(onPermissionChanged:) 既有呼叫）。

## Context Bundle

### 目標
在設定面板綁定區實作「新增綁定流程」與「偵測捕捉互動」（SPEC-007 FR-06 + FR-08 新增部分）。建構在 W4-003 已落地的面板結構上。FR-05 hotkey 實際合成執行屬 W5，本票只捕捉並存入 HotkeyAction。

### W4-003 已落地的面板結構（你的基礎，勿重造）
- lib/widgets/settings_panel.dart：`SettingsPanel` 為 StatefulWidget（State `_SettingsPanelState`），建構參數含 `required InputBindingController inputBindingController`（main.dart 經 _PanelHost 已接線）。
- 私有方法：`_buildBindingSection`（標題 + 權限列 + 清單，**新增綁定按鈕加在這裡**）/ `_buildPermissionRow` / `_buildBindingList` / `_buildBindingRow` / `_deleteBinding`（update+persist 即時持久化）/ `_actionSummary` / `_directionLabel`。
- 字串集中於 AppText（bindingSectionTitle / bindingListEmpty / bindingButtonPrefix / bindingActionDragScroll / bindingActionHotkey / bindingDirectionNatural/Inverted / bindingSensitivityPrefix / bindingKeyCodePrefix / bindingModifierPrefix / permission* 等）；新增字串續加 AppText。

### W4-001 已落地的 controller 捕捉 API
- `startButtonCapture({required ValueChanged<int> onCaptured, Duration timeout = AppDurations.buttonCaptureTimeout})`：請原生進捕捉，收到側鍵→onCaptured(buttonNumber) 後自動結束。
- `cancelButtonCapture()`：取消捕捉。
- **注意缺口**：timeout 時內部呼叫 cancelButtonCapture 結束捕捉，但**不通知 onCaptured**，UI 無從得知「逾時/取消而無結果」→ UI 會卡在 capturing 視覺。**請於本票補上**：建議擴充 startButtonCapture 加選用 `VoidCallback? onCancelled`（逾時與外部 cancel 時觸發），UI 用它重置 capturing 狀態。controller 為本票可改範圍。

### 模型（lib/input/）
- MouseBinding(buttonNumber, action)；DragScrollAction(direction: ScrollDirection.natural/inverted, sensitivity: double)；HotkeyAction(keyCode: int, modifiers: List<int>)。copyWith(bindings:) 會 dedupeBindingsByButton（同 buttonNumber 後者覆蓋）。

### 要實作（新增綁定流程）
1. _buildBindingSection 內加「新增綁定」按鈕，開啟新增流程（建議 inline 展開的 stateful 子表單；面板在 Stack overlay + MaterialApp 下 showDialog 亦可用，但 inline 較不依賴 Navigator，擇一，理由記 ticket）。
2. 流程階段：
   a. 偵測按鍵：呼叫 inputBindingController.startButtonCapture(onCaptured: (n)=>setState 選定 buttonNumber, onCancelled: ()=>setState 重置)。UI 顯示「請按下要綁定的滑鼠側鍵…」+ 取消鈕（→cancelButtonCapture）。
   b. 選動作型別：DragScroll / Hotkey（SegmentedButton 或 Dropdown）。
   c. 設參數：
      - DragScroll：方向（natural/inverted 切換）+ 靈敏度（Slider，預設 AppInputBinding.defaultDragScrollSensitivity）。
      - Hotkey：擷取鍵盤組合鍵——面板開啟時 window 已 focus（main.dart）；用 Focus/KeyboardListener/HardwareKeyboard 擷取 keyDown 的實體鍵碼 + 當下修飾鍵集合 → HotkeyAction(keyCode, modifiers)。UI 顯示捕捉到的組合。
   d. 確認：controller.update((s)=>s.copyWith(bindings: [...s.bindings, newBinding])) 後 persist()（即時持久化，與 W4-003 刪除一致）。取消則丟棄。
3. 逾時/取消：依上方 onCancelled 缺口處理，capturing 視覺可正確退出。

### 注意事項
- HotkeyAction.keyCode/modifiers 的 int 編碼：FR-05 原生 CGEvent 合成（W5）需 macOS virtual keycode；本票只負責「捕捉並存入」，Flutter 端擷取到的編碼與原生合成的對齊屬 W5 範圍。若擷取編碼來源不確定，存 Flutter 提供的實體鍵碼並於 Exit Status 標 NeedsContext 供 W5 處理，勿自行臆測映射。
- 重複 buttonNumber：copyWith dedupe 會以新綁定覆蓋同編號舊綁定（SPEC FR-01 允許）；UI 可選擇性提示，非必須。
- 捕捉需輔助使用授權；未授權時 startButtonCapture 的原生 tap 不建立（W4-002），UI 應引導先授權（W4-003 權限區已在）。

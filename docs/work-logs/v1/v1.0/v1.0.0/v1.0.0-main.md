## 前情提要

v0.x 完成後，screen_clock 已具備全螢幕透明遮罩 + 中央時鐘 + 多螢幕選擇。v1.0.0 把使用者體驗推到「可發布」水準：
- 從寫死預設值改為可設定（樣式、位置、目標螢幕、開機啟動）
- 從開發者工具改為打包好的 macOS app

Wave 安排：
- W1：設定持久化（資料模型 + 讀寫 + 啟動時套用）
- W2：設定面板 UI（呼出機制、動態 click-through、預覽）
- W3：開機啟動（LaunchAtLogin 整合）
- W4：收尾與發布（icon、README、code signing、打包、最終驗收）

待釐清的設計問題：
- W1 第一個 ANA：設定儲存方案（shared_preferences vs 檔案）
- W2 第一個 ANA：設定面板呼出機制（右鍵 menu / 鍵盤快捷鍵 / 系統 menu bar）
- W3 第一個 ANA：macOS LaunchAtLogin 方案（LaunchAtLogin 套件 vs 手寫 launchd plist）

#### 最終驗收程序（W4-005）

> 預期環境：macOS 12+；已安裝 Xcode（用於簽署）；已準備 Apple Developer ID + App-specific password。

##### A. 建置與簽署

```bash
# Apple Developer 帳號 / 證書設定到環境變數
export DEVELOPER_ID='Developer ID Application: Your Name (TEAMID)'
export APPLE_ID='you@example.com'
export APPLE_APP_PASSWORD='xxxx-xxxx-xxxx-xxxx'
export APPLE_TEAM_ID='XXXXXXXXXX'

# 全自動 build + sign + notarize + dmg
scripts/build-release.sh 1.0.0

# 輸出：dist/screen_clock-1.0.0.dmg
```

如果只想做 dev build 跳過 notarize：

```bash
SKIP_NOTARIZE=1 scripts/build-release.sh 1.0.0
```

##### B. 安裝與 Gatekeeper 驗證

```bash
spctl --assess --type execute --verbose=4 dist/screen_clock.app
# 預期: dist/screen_clock.app: accepted source=Notarized Developer ID
```

| # | 動作 | 通過標準 |
|---|------|---------|
| B1 | 開 dmg 拖到 Applications | 無 Gatekeeper 警示 |
| B2 | 首次啟動 | 不被 macOS 攔截 |
| B3 | App 出現在 Launchpad | 圖示正常（W4-001 替換後） |

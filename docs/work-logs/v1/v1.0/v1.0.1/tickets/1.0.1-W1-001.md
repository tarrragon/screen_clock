## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 症狀
flutter run -d macos 啟動後，透明遮罩視窗顯示為黑色背景，遮擋桌面底下內容，違反 SPEC-001 FR-02「背景真透明」與遮罩時鐘核心需求。

### 根因
macOS 原生端只把 NSWindow（self）設為透明，漏設 FlutterViewController 透明層。

現況 macos/Runner/MainFlutterWindow.swift（已正確）：
- self.isOpaque = false（line 13）
- self.backgroundColor = .clear（line 14）

Dart 端三處透明（已正確）：
- lib/main.dart:55 windowManager.setBackgroundColor(overlayBackground)
- lib/main.dart:173 Scaffold backgroundColor: overlayBackground
- lib/app_constants.dart:48 overlayBackground = Colors.transparent

缺失：FlutterViewController 本身的 backgroundColor 未設。Flutter 3.7+ macOS embedder 的 FlutterView backing layer 預設 opaque，會自繪黑底蓋在透明 NSWindow 上。只設 NSWindow 透明不足，必須同時設 flutterViewController.backgroundColor = .clear。

### 證據
- flutter/flutter #119132（Flutter 3.7.0 renders black background on macOS）
- leanflutter/window_manager #293（macOS transparency broken in Flutter 3.7.0）
- flutter/flutter #59969（Transparent FlutterViewController）
- 本機環境：Flutter 3.38.10 / Dart 3.10.9

### 關鍵結論
此黑底在 Flutter 3.7+ 持續存在，升級 Flutter 不會自動修復，必須補原生設定。


### WRAP 行前三問
- **W（其他做法）**：(1) 補設 `flutterViewController.backgroundColor = .clear`【選此，flutter/flutter #119132 等官方 issue 證實的標準解】；(2) 調整 `setBackgroundColor` 色值【無效，根因在 FlutterView backing layer 非 NSWindow】；(3) 升級 Flutter【無效，regression 持續存在於 3.7+】。
- **A（機會成本）**：單檔一行原生設定，不擠壓其他 v1.0.x 目標；屬遮罩核心需求 hotfix。
- **P（最可能失敗）**：原生改對但本機 build 快取未清導致視覺仍黑底；防護：`flutter clean` 後重 build 再目視驗收。
## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
### 變更摘要

macos/Runner/MainFlutterWindow.swift 於既有透明設定區塊（self.isOpaque / self.backgroundColor）後補一行：

```swift
flutterViewController.backgroundColor = .clear
```

僅動此 Swift 檔，未改 Dart 端（lib/ 透明設定已正確）。

### 註解設計

於新增行上方補兩行業務情境註解，說明根因與必要性：FlutterView backing layer 在 Flutter 3.7+ macOS 預設不透明，會自繪黑底蓋住透明 NSWindow，必須與 NSWindow 同步清背景，否則透明遮罩顯示為黑底（引用 flutter/flutter #119132）。註解聚焦「為何需要」而非「做了什麼」，與既有 SPEC-001 FR-02 註解同群組。

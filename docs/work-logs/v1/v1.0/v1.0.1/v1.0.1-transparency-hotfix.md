## 進度追蹤

- 2026-05-29: [建立] -- 建立 v1.0.1 版本與 ticket 1.0.1-W1-001（透明黑底 hotfix）
- 2026-05-29: [附帶處理] -- 將 SDK 約束放寬（pubspec ^3.11.1 → ^3.10.0，相容本機 Flutter 3.38.10）獨立 commit 並 merge 回 main（47a33f2）
- 2026-05-29: [維護] -- untrack runtime state 並補 gitignore，解除 PC-019 worktree 派發阻擋（25dcd7e）
- 2026-05-29: [完成] -- 1.0.1-W1-001 完成：MainFlutterWindow.swift 補設 `flutterViewController.backgroundColor = .clear`（62b8991）。flutter test 29 passed，GUI 三項視覺驗收（背景透明 / click-through / always-on-top）全通過

## 關鍵決策

| 決策 | 結論 | 理由 |
|------|------|------|
| 分支策略 | SDK 放寬獨立 merge 回 main，hotfix 另開 fix 分支 | 黑底是設定問題、與版本無關，兩者應分離追蹤 |
| 根因 | FlutterView backing layer 在 Flutter 3.7+ macOS 預設不透明 | 只設 NSWindow 透明不足，須同步設 FlutterViewController 透明（flutter/flutter #119132） |

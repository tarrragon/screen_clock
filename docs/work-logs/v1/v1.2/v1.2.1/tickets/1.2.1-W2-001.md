## Problem Analysis
<!-- Schema[IMP/Problem Analysis]: 選填（小型 IMP 可留 placeholder；大型 IMP 建議填寫）（.claude/pm-rules/ticket-body-schema.md） -->
### 問題根因
v1.2.0 的 `visibleOnFullScreen=false`（fullScreenAuxiliary）僅對 native fullscreen（綠燈鈕、獨立 Space）有效。假全螢幕（YouTube 全螢幕 / Keynote 播放 / 遊戲）是鋪滿螢幕的「普通視窗」，不建獨立 Space，該設定技術上無法讓時鐘讓位（ANA 1.2.1-W1-001 已附 window_manager 原生碼證據）。

### 影響範圍
- macos/Runner/MainFlutterWindow.swift（新增原生 CGWindowList 覆蓋偵測）
- lib/main.dart（接收覆蓋狀態，切換時鐘渲染）
- lib/app_constants.dart（channel 名 / 方法名 / 參數鍵常數集中）
- lib/platform/fullscreen_detector.dart（新增 Dart 橋接服務）

### 相關 Error Pattern
PC-165（測試綠燈 ≠ runtime 正確）：原生 CGWindowList 覆蓋判定屬 macOS 環境行為，Dart widget/unit test 無法證明，故 acceptance #3 保留實機 runtime 驗收。

## Solution
<!-- Schema[IMP/Solution]: 選填（.claude/pm-rules/ticket-body-schema.md） -->
依 ANA 推薦方案 A（CGWindowList 覆蓋判定）為主 + C（NSWorkspace 通知觸發源）+ B（前置過濾）。

### 原生端（macos/Runner/MainFlutterWindow.swift）
新增 `FullscreenCoverageDetector`：
- 觸發源（方案 C）：訂閱 `activeSpaceDidChangeNotification` / `didActivateApplicationNotification` / `didChangeScreenParametersNotification`；觸發後延遲 0.3s 再評估（等視窗清單穩定）。
- 覆蓋判定（方案 A）：`CGWindowListCopyWindowInfo([.optionOnScreenOnly, .excludeDesktopElements])`，排除本 app PID（自身遮罩視窗）、僅看 layer==0（normal，排除 Dock/menubar），判斷是否有視窗鋪滿遮罩視窗所在螢幕。
- 座標轉換：NSScreen frame（左下原點）翻轉為 CG 座標（左上原點）對齊 CGWindowList。
- 只讀 bounds + layer + owner PID（不取 window title），不需 Screen Recording 權限。
- 僅在狀態變化時 invokeMethod 回報（lastCovered 去抖）。
- 層級 1 驗證：coverage changed 時 NSLog 印出 covered 與 screenFrame。

### Dart 端
- lib/platform/fullscreen_detector.dart：`FullscreenDetector` 透過 method channel 接收原生回報，解析 covered 旗標（型別不符保守視為未覆蓋），轉成 callback；MethodChannel 可注入便於測試。
- lib/main.dart：新增 `_coveredByFullscreen` 狀態，與使用者手動意願 `_clockVisible` 以 `_shouldRenderClock = _clockVisible && !_coveredByFullscreen` 合成，條件渲染 CenterClock。沿用 v1.2.0「不動原生視窗」隱藏機制。手動隱藏與假全螢幕讓位互不干擾。
- lib/app_constants.dart：新增 `AppFullscreenDetect`（channelName / onCoverageChangedMethod / coveredArgKey），禁硬編碼。

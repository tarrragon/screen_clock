# v0.3.0 版本工作日誌

**版本號**: v0.3.0
**開始日期**: 待 v0.2.0 完成
**目標**: 多螢幕選擇能力（指定目標螢幕、處理熱插拔）。
**狀態**: planning

---

## 前情提要

v0.1.0 與 v0.2.0 完成後，screen_clock 可在主螢幕運作。本版本允許使用者指定遮罩顯示在哪一個螢幕。

MVP 版本不含設定 UI，只接受 CLI 引數（如 `--screen=1`）。設定 UI 留待 v1.0.0。

待釐清的設計問題（W1 第一個 ANA ticket）：
- 螢幕偵測：`screen_retriever` 套件 vs `window_manager` 自帶 API
- 螢幕識別：Display ID 是否在拔插後保持穩定

#### 已知限制

- `Display` 不提供絕對 position，僅 `visiblePosition`（排除 menu bar）。次要螢幕上 menu bar 區域可能不被遮罩覆蓋。若 W1-006 驗收觀察到此問題明顯，可後續以平台原生程式碼補絕對位置。
- 新增螢幕不會主動切換目標（SPEC-003 設計約束）；要切到新螢幕需以 `--screen=N` 重啟。

#### 手動驗收程序（W1-006）

> 預期環境：macOS 12+；連接至少 2 個螢幕（一個外接）。

##### A. 單螢幕 baseline

```bash
flutter run -d macos --release
```

| # | 項目 | 通過標準 |
|---|------|---------|
| A1 | 啟動無 `--screen` 引數，遮罩出現在主螢幕 | 與 v0.1.0 + v0.2.0 行為一致 |
| A2 | `flutter run -d macos --release -a --screen=99` 仍顯示主螢幕並 log warning | stderr 有 `out of range; fallback to primary` |
| A3 | `--screen=abc` 仍顯示主螢幕並 log warning | parseScreenArg 返 null，等同 A1 |

##### B. 雙螢幕

| # | 項目 | 通過標準 |
|---|------|---------|
| B1 | `--screen=0` 顯示在主螢幕 | 同 A1 |
| B2 | `--screen=1` 顯示在第二螢幕 | 視窗左上對齊 visiblePosition |
| B3 | 第二螢幕上時鐘維持中央，click-through 維持 | SPEC-002 + SPEC-001 在新螢幕仍生效 |
| B4 | 已知限制驗證：B2 情境下 menu bar 是否被遮蓋 | 紀錄結果（若擋住記為 pass、若未擋住記為 known limitation） |

##### C. 熱插拔

| # | 動作 | 通過標準 |
|---|------|---------|
| C1 | B2 啟動後拔除第二螢幕 | 1 秒內視窗回到主螢幕（FR-05 + NFR-01） |
| C2 | 重新插回第二螢幕 | 視窗保持在主螢幕（不主動切回） |
| C3 | A1 啟動後接入第二螢幕 | 視窗保持在主螢幕、log 顯示 event 觀察到 |

回歸：

| # | 動作 | 預期 |
|---|------|------|
| R1 | 暫時把 `_safePrimary` 拋例外（測試）→ 啟動 | 視窗使用 `AppSizes.fallbackWindowSize`，不 crash |
| R2 | 模擬 `getAllDisplays` 回傳空 → 啟動 | 等同 R1 |

若有 fail，於當日新增段記錄問題、影響、follow-up ticket。

# v0.1.0 版本工作日誌

**版本號**: v0.1.0
**開始日期**: 2026-05-29
**目標**: macOS 全螢幕透明遮罩視窗（frameless / always-on-top / click-through / no-shadow）覆蓋 SPEC-001。
**狀態**: planning

---

## 前情提要

PROP-001 已通過 draft 狀態，target_version 指向本版本。

SPEC-001 已凍結（draft），定義 6 個 FR + 2 個 NFR：
- FR-01 全螢幕覆蓋主螢幕
- FR-02 真透明背景
- FR-03 無邊框、無陰影
- FR-04 永遠置頂
- FR-05 滑鼠 click-through
- FR-06 鍵盤焦點不被攔截
- NFR-01 啟動 < 1500ms
- NFR-02 macOS-only（Windows 在 v1.1.x）

本版本不含 UI 內容（時鐘 widget 在 v0.2.0）。完成後應能看到「全螢幕透明遮罩 + 可穿透」的純框架。

#### 手動驗收程序（W1-005）

> 預期環境：macOS 12+；連接至少一個顯示器；無其他 screen_clock instance 在執行。

啟動：

```bash
flutter run -d macos --release
```

依下列順序驗證；每項用一句結論（pass / fail / 看到什麼）寫回本日誌追加段。

| # | 項目 | 對應 SPEC / UC | 通過標準 |
|---|------|----------------|---------|
| 1 | 啟動後 1.5 秒內看到全螢幕遮罩 | SPEC-001 NFR-01 / UC-01 主場景 | 視窗在 release build 1500ms 內可見 |
| 2 | 視窗背景看得到底下桌布 | SPEC-001 FR-02 / UC-01 驗收 | 肉眼確認桌布、Dock、其他視窗都可見 |
| 3 | 視窗無邊框、無紅黃綠按鈕 | SPEC-001 FR-03 | 視窗邊緣完全乾淨 |
| 4 | 視窗無陰影 | SPEC-001 FR-03 | 邊緣無灰邊殘留 |
| 5 | 點擊遮罩任何位置都命中底下 app | SPEC-001 FR-05 / UC-02 主場景 1 | 點 Finder、Dock、其他視窗都正常作用 |
| 6 | 從 Finder 拖檔到底下 app 可成功 | UC-02 主場景 2 | 拖曳軌跡穿過遮罩、放下成功 |
| 7 | 在底下 app 打字鍵盤不被擋 | SPEC-001 FR-06 / UC-02 主場景 3 | 文字正確輸入到底下 app |
| 8 | trackpad/滾輪在遮罩上捲動底下內容 | UC-02 主場景 4 | 底下視窗正常捲動 |
| 9 | 開啟其他視窗後遮罩仍在最上層 | SPEC-001 FR-04 | 切換多個 app 都看得到時鐘區 |
| 10 | Cmd+Q 1 秒內結束 process | UC-03 驗收 | Activity Monitor 確認無殘留 |
| 11 | 啟動後 5 分鐘閒置 CPU 平均 < 1% | NFR-01 / NFR 量測 | M1+ 量測 |

回歸案例（測試 SPEC-001 設計約束）：

| # | 動作 | 預期 |
|---|------|------|
| R1 | 暫時把 `MainFlutterWindow.swift` 的 `isOpaque = false` 改成 `true` | 視窗變白底（驗證 Swift 設定為透明必要條件） |
| R2 | 主螢幕解析度切換後重啟 app | 視窗貼合新尺寸 |
| R3 | Mission Control 切換 | 退出後遮罩自動回到原狀態 |

若有項目 fail，在當日日期下追加段記錄問題、影響與 follow-up ticket ID。
